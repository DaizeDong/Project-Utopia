> **MVP from HW02:**
>
> The hardest technical challenge is integrating an LLM-based AI agent that issues meaningful, schema-validated decisions for NPC roles and environment policy in real time, with a deterministic fallback when the API is unavailable.
> The mid-term demo proves this works by showing AI-driven NPCs visibly change behavior in response to a live LLM decision, with the fallback mode keeping the simulation stable when the API is disabled.

> Everything else-multiple NPC role types, map editing polish, performance HUD, and scenario presets-is cut to keep the AI integration solid.

## AI Utility

AI has helped in almost all aspects of the project, from high-level design decisions to runtime behavior:

1. Structure Design: Codex helped generate modular code structure for AI integration and fallback handling.
2. Coding Assistance: Codex helped write most of the codebase and algorithms, including A* pathfinding and Boids steering.
3. Prompt Engineering: Codex helped design effective prompts for both environment and NPC policy decisions when using LLM as agents.
4. Runtime Debugging: AI-generated runtime traces and health checks made it easier to diagnose issues.

## Roadmap to Final

1. Refine the behavior and decision-making of the LLM-driven NPCs to make their actions more rational,
2. Refine the design of NPC decision state transitions, making them more responsive to the environment and policy changes and make things mode real.
3. Add more terrain types, buildings, NPCs.
4. Strengthen the interactions between different NPCs, making them more dynamic and emergent.
5. Continue optimizing performance, especially for larger maps and more entities.

## **Assignment 1: Group Formation, Pillars, and Conceptualization**

### **1\. Course Project Overview**

The core of this course is a semester-long, group-based project. Your objective is to incrementally develop an **end-to-end interactive visual experience**. This project should demonstrate a deep understanding of computer graphics and visual computing by applying theoretical concepts to a functional, real-time application.

### **2\. Group Formation Guidelines**

* **Duration:** Groups formed now will remain together for the entire duration of the course.  
* **Size:** The recommended group size is **2 to 4 students**.  
* **Exceptions:** Groups larger than 4 require a formal written justification submitted to the instructor, detailing why the project scope necessitates additional members and how roles will be distributed.  
* **Leadership:** Each group must designate one **Group Leader**. This person will be responsible for all official project submissions and will serve as the primary point of contact for the teaching staff.  
* **Please create your group on Canvas** 

### **3\. The Pillars of Computer Graphics & Visual Computing**

Your project must be built upon at least **two** of the following fundamental pillars. Use these categories to define the technical scope of your interactive experience:

| Pillar | Description | Example Application |
| :---- | :---- | :---- |
| **Rendering** | The process of generating an image from a 2D or 3D model. Includes lighting, shading, texturing, and ray tracing. | Implementing a custom PBR (Physically Based Rendering) shader or a real-time ray-traced shadow system. |
| **Animation** | Creating the illusion of motion. Includes keyframing, skeletal animation, inverse kinematics (IK), and vertex displacement. | A character system with procedural walking animations or complex skeletal transformations. |
| **Physics & Dynamics** | Simulating physical systems. Includes rigid body dynamics, soft body simulation, cloth, and fluids. | A water simulation environment or a destructible terrain system based on rigid body collisions. |
| **Collision Detection** | Calculating when two or more objects intersect and how they respond to that contact. | A fast-paced projectile system or a complex environment where objects must not overlap. |
| **Pathfinding & Navigation** | Algorithms for moving entities through an environment while avoiding obstacles. | A strategy game where units find the shortest path through a dynamic maze (e.g., A\* or Dijkstra). |
| **NPC AI / Behavior Trees** | Simulating "intelligent" behavior in non-player characters using state machines or behavior trees. | An ecosystem simulation where predators hunt prey based on visual cues and energy levels. |
| **Swarm & Boids Modeling** | Simulating the collective behavior of large groups (flocking, schooling, herding). | A massive starship battle with hundreds of independent fighters or a realistic school of fish. |
| **Procedural Generation** | Using algorithms to create data (geometry, textures, worlds) rather than creating it manually. | A game with an infinitely generating terrain or a forest where every tree is uniquely grown via L-systems. |

### **4\. Example Project Ideas**

* **"The Deep Sea Explorer":** (Pillars: *Fluid Physics, Procedural Generation*)  
  * A submarine simulation where the terrain is procedurally generated as the player dives deeper. Includes a real-time fluid simulation for bubbles and debris.  
* **"Swarm Commander":** (Pillars: *Swarm Modeling, Pathfinding*)  
  * An interactive visualizer where the user directs a swarm of 1,000+ drones through a moving obstacle course using Boids algorithms and dynamic pathfinding.  
* **"The Interactive Solar System":** (Pillars: *Rendering, Physics*)  
  * A high-fidelity space visualizer focusing on custom atmospheric scattering shaders (Rendering) and N-body gravity simulations (Physics).

### **5\. Assignment 1 Deliverables**

The Group Leader must submit the following: 

1. **Group Roster:** Full names and NetIDs of all members. Clearly mark the **Group Leader**.  
2. **Selected Pillars:** List the 2+ pillars your project will focus on.  
3. **Project Concept Summary:** A 250-500 word description of the "Interactive Visual Experience." What will the user do? What is the visual goal?  
4. **AI-Generated Mockups:** Use Gemini or another AI tool  to create visual representations of your concept.   
5. **Interactive Mockup (Optional but Recommended):** A simple web-based prototype (HTML/JS/CSS) generated with AI to show basic concept.

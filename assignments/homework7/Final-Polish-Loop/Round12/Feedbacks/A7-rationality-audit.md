# A7 - Rationality Audit - Round 12 (Tier A, R12)

- **Build URL:** http://127.0.0.1:5173/
- **Date:** 2026-05-02
- **Mode:** Strictly blind. No prior R12 feedback consulted before/while reviewing. No source files read for in-game labels — every defect was observed in the live DOM via Playwright.
- **Coverage:** Title screen (briefing, BEST RUNS, controls), in-game (1920x1080 + 1024x768), Build / Colony / Settings / AI Log / Debug sidebar tabs, Help modal (Controls / Resource Chain / Threat & Prosperity tabs), heat lens (L), terrain overlay cycle (T x4), pause (Space), worker inspector (`Ula Venn`).

## Verdict

**Conditional Pass — Director Rationality Fail / Label Consistency Fail / Engineering Leakage Fail.** The R3 closures (heat-lens supply-surplus context label, autopilot Goal-reached cap, Threat scale anchor) are still in place at runtime. The new `Threat 33% (raid at 80%)` HUD chip and the Inspector `Food Diagnosis` line both render and are useful. **However**, three structural rationality issues stack on every fresh run with **no player intervention required**: (1) the Colony tab still flashes a green **STABLE** badge while showing `Food: -151/min` and `1m 12s until empty` and `farms 0/6 | warehouses 0/2`, with the Director's natural-language headline naming **"rebuild the broken supply lane"** as the next move while food is in free-fall; (2) the AI Decisions block emits glued-token strings (`saboteursstrike`, `workersrebuild`, `tradersshug`) that read like a code regression, not narrative; (3) engineering telemetry leaks into player-facing surfaces in three independent places (`Why no WHISPER?: LLM never reached` permanent header chip; `proxy=unknown model=gpt-5-4-nano` in AI Log; `reachable=unknown source=unknown sampling…` strings in Inspector / Resources rows).

## Top 3 Rationality Issues (Dead / Contradictory / Misaligned)

1. **(Director / HUD contradiction — P0) Colony tab flags `STABLE` while the colony is 72 seconds from food collapse, and the AI headline cites the wrong crisis.** `screenshots/A7/13-colony-tab.png`: top-of-Colony reads `STABLE` (green pill) on Day 3, with `Food: -151/min · 0 idle · Threat 33% (raid at 80%)` on the same line and `Food 179 ▼ -151.1/min (cons -148, spoil -3) · ► 1m 12s until empty` (red urgency cue) one row below. Population panel under the same tab reports `Workers 12` with role split `FARM 7 / WOOD 5 / STONE 0 / HERBS 0 / COOK 0 / SMITH 0 / HERBALIST 0 / HAUL 0`, while the Inspector for `Ula Venn (FARM)` reads `Food Diagnosis: Food exists, but there is no warehouse access point` and the `Food Route Facts` row underneath confirms `warehouses=0, farms=0`. Seven workers carry the FARM **role** while zero farms exist — a dead role assignment that produces no food, deepening the cliff. Above all that, the AI STORYTELLER line still reads `DIRECTOR picks rebuild the broken supply lane: Reconnect the lumber line before the east depot runs dry — the plains forgive distance, not silence.` with `Live Causal Chain → Restore west lumber route` (`15-ai-log.png`) — both surfaces fixated on the **lumber** restoration path while the actual T+72s failure path is **food**. Result: a new player reading the Colony tab is told the colony is STABLE and the next move is a road, while the simulation will starve them out before they finish placing it. Fix: (a) gate `STABLE` on `food_runway_minutes > raid_warning_threshold` AND `farms > 0` AND `warehouses > 0`; the current state should read `STARVING` or `CRITICAL`; (b) re-order Director priority queue so `food_runway < 5min` preempts logistics narratives; (c) hide the FARM role count when `farms==0`, or relabel the role chip as `FARM (idle - no farm)`.

2. **(Text / AI surface — P1) AI Decisions block emits glued-token narrative strings that read as a build regression.** `screenshots/A7/15-ai-log.png` and `16-debug.png` — the AI summary line under `Decision Results` reads literally `AI: env=let the colony breathe | saboteursstrike a soft frontier corridor | workersrebuild the broken supply lane | tradersshug the warehouse lanes`. Three concatenations (`saboteurs`+`strike`, `workers`+`rebuild`, `traders`+`shug` [almost certainly `traders`+`hug`]) are missing the space between role and verb. The same string is rendered verbatim in three places — Live Causal Chain (AI Log), AI Decisions (Debug → System Narrative → AI), and the Director timeline tooltip — so the same template authoring bug surfaces three times per tick. This is the highest-confidence dead-text defect in the run because it is the AI's primary storytelling surface and the prose breaks rationality on first glance. Fix: locate the template join (probably a `[role, verb].join('')` instead of `.join(' ')` or `${role}${verb}`) and add the space; verify against `policy.subjects`.

3. **(Engineering leakage — P1) Three independent debug strings render permanently in player-facing UI, all phrasing the same fact in different jargon.** (a) Top-of-screen, every in-game frame: `Why no WHISPER?: LLM never reached` (`02-game-start.png`, `06-heat-lens.png`, `13-colony-tab.png`); (b) AI Log header: `Autopilot OFF (LLM calls disabled; fallback directors still visible) coverage=fallback mode=fallback proxy=unknown model=gpt-5-4-nano` (`15-ai-log.png`); (c) Debug → World State: `AI Mode: off / fallback (unknown, gpt-5-4-nano)` and `AI Env: fallback @ 216.6s | let the colony breathe`. The same fact ("the LLM proxy is offline; rule-based directors are running") is conveyed by `WHISPER`, `coverage=fallback`, `proxy=unknown`, `model=gpt-5-4-nano`, and `(unknown, gpt-5-4-nano)`. None of these mean anything to a player who hasn't read the source. Worse, the AI Mode flips between `(unknown, ...)` (Debug, 1920) and `(up, ...)` (Debug, 1024 — `17-1024-debug.png`) for the same offline state, contradicting the "LLM never reached" header chip. Fix: (a) replace top-chip with a single icon `[AI offline]` + tooltip; (b) collapse AI Log header to plain `Autopilot OFF — LLM proxy offline; rule-based fallback directors steering`; (c) drop `proxy=unknown` and `coverage=fallback` from any user-facing surface or move them under a Debug-only "Show telemetry" toggle; (d) reconcile `unknown`/`up`/`never reached` to one canonical state name.

## Secondary Rationality Defects (not in top 3, all confirmed in screenshots)

- **Hotkey range contradiction across three surfaces.** Title-screen briefing chip says `1-12 tools` (`01-title.png`); in-game help modal says `1 - 12 - quick-pick build tool (12 tools in the Build toolbar; hover any button for name + hotkey)` AND in the same modal says `Press 1 - 9 to quick-pick a build tool`; the in-game keymap legend says `1-9 / - / =` (`02-game-start.png`, `06-heat-lens.png`). Three different ranges for the same set of buttons. The actual toolbar exposes 12 buttons hot-keyed `0,1-9,-,=` (the disabled `Clinic` carries no key prefix). Suggest standardising on the literal keys: `0,1-9,-,=` (12 keys) and dropping the misleading `1-12` fiction.
- **Building/role naming drift Help vs toolbar.** Help (Resource Chain tab, `04-help-resource-chain.png`) names buildings `Lumber Camps / Quarries / Herb Gardens`; the Build toolbar (`02-game-start.png`) names them `Lumber / Quarry / Herbs`. Inspector role chips read `WOOD / STONE / HERBS` (`11-worker-inspect.png`). Three vocabularies for the same chain.
- **Hunger labels are a four-bucket scale with no legend, and `(44% fed)` is a fifth.** Inspector says `Hungry (44% fed)`; the worker row for the same `Ula Venn` says `Hungry / FARM ... hungry` (lowercase appended to verbose label); other rows show `a bit hungry` and `well-fed`. Filter pills above the entity list slice the same scale as `Critical hunger 0 / Hungry 1 / ...` — the boundary between `Critical hunger`, `Hungry`, `a bit hungry`, and `well-fed` is never displayed and `(44% fed)` is the only number shown. Suggest hover-tooltip on each pill: `Critical < 0.20 / Hungry 0.20-0.45 / Bit hungry 0.45-0.70 / Well-fed >= 0.70`.
- **Filter pill counts do not always sum.** First-frame snapshot: `My workers 12 / All 25 / Critical 0 / Hungry 1 / Blocked 11 / Working 1 / Idle 10 / Hauling 0 / Combat 2 / Other 0` — Blocked+Working+Idle+Hauling+Combat+Other = 24 vs `My workers 12` and `All 25`. By T=15s: `Combat 4 / Idle 8 / Working 1 / Blocked 8 / Hungry 2 / All 22` -> 8+1+8+0+4+0=21 vs 22. By T=03:42: `My 12 / All 22 / Combat 5 / Other 1 / Idle 6 / Working 1 / Blocked 3 / Hungry 5` -> sums to 16 vs 22. Either categories overlap (need to disclose) or six workers are uncounted (needs an `Uncounted` bucket). Pills currently look like a partition but aren't.
- **`east ruined depot x2` floating tile label is a count-as-suffix construction that reads as a typo.** `02-game-start.png`. The `x2` after a singular noun phrase is grammatically weird; suggest `2 ruined depots (east)` or just badge a numeric icon on the tile.
- **Death attribution string `killed-by-worker` contradicts the Debug Deaths counter.** Toast `Bear-25 died - killed-by-worker` (`06-heat-lens.png`); Debug → Deaths `8 (starve 0 / pred 8)` (`16-debug.png`). The Deaths breakdown only exposes `starve` and `pred`, so a worker killing a bear must be counted under `pred` (predator-related) — but the toast told the player a worker did it. Either the bucket should be `(starve / pred / hunt)` to match attribution, or the toast should say `killed by guard` (the actual `setWorkerRole` was GUARD, not worker, in every visible row).
- **Pause toast is the only signal Space worked, and it's a small bottom-left pill that fades.** `10-paused.png`. Confirmed via `evaluate({frozen: true})` that the timer is genuinely frozen, but there is no canvas-wide affordance — no `PAUSED` overlay, no toolbar pause button pressed-state, no border tint. (Also flagged by A6, including here for cross-check because rationality + state-feedback overlap.)
- **Terrain overlay shows two labels for the same state.** `07-overlay-T1.png` shows `Overlay: Fertility` top-left badge AND `Terrain Overlay: Fertility overlay ON.` bottom-left toast. Same fact, two surfaces, two phrasings, no legend mapping color -> value. Connectivity overlay (`09-overlay-T3.png`) is mostly uniform yellow over land — without a legend the player cannot tell which tiles are connected to what.
- **Inspector "Click a worker..." tooltip persists after a worker is selected.** `11-worker-inspect.png`, `12-food-route-facts.png` — bottom of the Entity Focus list still shows `Click a worker, visitor, or animal on the map to inspect it here` while a full inspector card is rendered for `Ula Venn` two inches below. Should hide once a target is bound.
- **Inspector `State: Wander | Intent:` — Intent value is empty across every captured frame.** Either remove the dead `Intent:` label or populate it from `worker.fsm.target.kind` (per CLAUDE.md the FSM target field exists).
- **`Group: workers` row in inspector is dead-redundant** — already implied by `Type: WORKER` and `Role: FARM`.
- **`Food Route Facts: source=unknown, reachable=unknown, sampling...` is debug telemetry in player UI** — `12-food-route-facts.png`. Should resolve to `source: stockpile (unreachable)` and `reachable: no` rather than `unknown`.
- **AI Log → AI Decisions section claims `predators:isolated prey patrol`** while Population shows `Predators: 0` (`13-colony-tab.png` vs `15-ai-log.png`). The fallback director is reciting an empty narrative.
- **Top-bar `Wood: 34` shows arrow `·` not `▼`/`▲`** while Food shows `▼` — inconsistent direction-glyph contract across the resource chips.
- **Resources rows in Colony tab show `(sampling...)` for Wood/Stone/Herbs/Meals/Tools/Medicine** (`13-colony-tab.png`) — this is internal "warming up rate window" jargon; should display `0/min` or `--` until ready, not the engineering verb.
- **`Run 00:03:52 Score 152 Dev 24/100 — Scrappy outpost, still finding its rhyt` ellipsis** at 1024 viewport (`17-1024-debug.png`) and at 1920 viewport (`18-heat-cycle2.png`) — top scoreboard label is constantly clipped mid-word, hiding both the score description and confirmation that Dev/100 is being tracked. (Cross-flagged with A6.)
- **The "plus" badge with a blue dot near the `Heat (L)` vertical tab** (`06-heat-lens.png`, `18-heat-cycle2.png`) is half-clipped against the right rail and has no aria-label or tooltip — it reads as a notification badge but flips state silently across L cycles. (Cross-flagged with A4 / A6.)
- **Pause toast at 1024 wraps to two lines** (`Story AI is offline -- fallback director is steering. (Game still works.) [timeout]`) — the `[timeout]` debug suffix is leaked into the player toast (`17-1024-debug.png`). Drop the bracketed reason or move it into a Debug-only details row.
- **Dev/Threat/Prosperity scales are inconsistent.** Top HUD: `Dev 24/100`. Colony tab: `Threat 33% (raid at 80%)`. Debug World State: `Prosperity 22.3 / Threat 17.8` (raw decimals, no /100). Three numeric scales for three reputation-style metrics; suggest standardise on `n/100` for all three and stop showing raw `Threat 17.8` in Debug as if it were a different metric.
- **`Story AI is offline ... (Game still works.) [timeout]` toast contradicts the AI Log header which calls it `LLM calls disabled` (player chose), not a timeout** (`13-colony-tab.png` vs `15-ai-log.png`). If the LLM call truly never went out (`LLM never reached` header chip), it isn't a `[timeout]` — it never started.

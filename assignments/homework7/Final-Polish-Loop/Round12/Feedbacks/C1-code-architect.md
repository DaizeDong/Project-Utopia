## New debt from R11 commits

- **Halo mesh capacity ceilings duplicate sphere mesh ceilings** (PGG-sphere) — `maxWorkers=1200`, `maxVisitors=240`, `maxHerbivores=300`, `maxPredators` are declared once for sphere meshes and re-read for halos. Refactor opportunity: hoist to `this._entityCapacities = {workers: 1200, ...}` so a future capacity bump touches one site, not eight.
- **Trail buffer sized at 1200 × 8 × 2 × 7 floats = 134,400 entries (~538 KB)** (PHH) — pre-allocated for max-population worst case, never resized. At typical 200-worker mid-game runs, ~83% of the buffer is zero-padded each frame. Acceptable for now, but worth a TODO if the cap is ever raised.
- **EWMA reverse-map `Int32Array(W*H)` is rebuilt on every `#rebuildTilesIfNeeded` call** (PHH) — fine because rebuild is rare, but the `.fill(-1)` + nested-loop set is `O(W*H)`. If W or H ever exceeds 96×72 the cost climbs linearly. Single-line cost — flag, don't fix.
- **A1 validation walks MAP_TEMPLATES via `.some()` on every regen call** (A1) — O(templates). Five templates today; trivial. Worth converting to a Set memoized at module load if MAP_TEMPLATES grows past 20.
- **PII tracks two new instance fields without underscore convention break** — `_llmLastModeCombined`, `_llmDegradeLastEmitSec`. Consistent with R10's `_eventDrainBudgetTick`/`_zombieSinceSec` etc. — good. Now seven underscored private fields in HUDController; not yet at refactor threshold but trending.
- No new globals, no new circular imports, no new direct DOM access in simulation. PII's `document.querySelector(".overlay-panel.run-ended")` IS a new DOM access in `GameStateOverlay` — but GameStateOverlay is a UI class so this is in-domain, not a layering violation.

# **Assignment 4: Production Planning & MVP 2.0 (Alpha)**

## **1\. Overview**

You have successfully demonstrated your core technical pillar in Assignment 3\. Now, you must turn that "tech demo" into a cohesive interactive experience.

Assignment 4 has two primary goals:

1. **Project Management:** Map out exactly how you will finish the project by the final deadline.  
2. **MVP 2.0 (The Alpha Build):** Integrate your core mechanic with the necessary "wrapper" (UI, controls, game loop) to make it a complete software product.

## **2\. Phase 1: The Production Schedule**

Based on your progress and feedback, you must create a detailed **Weekly Development Plan** for the remainder of the semester.

**Requirement:** Create a week-by-week breakdown (e.g., Weeks 9–14) covering:

* **Engineering Tasks:** (e.g., "Implement collision detection for enemy types," "Optimize shader for mobile").  
* **Asset Tasks:** (e.g., "Generate final skybox using AI," "Model main character").  
* **Integration:** When do the separate parts come together?  
* **The "Buffer Week":** You **must** dedicate the final week exclusively to bug fixing, documentation, and polish. **Do not plan new features for the final week.**

*Format:* A Table or Gantt Chart included in your PDF report.

## **3\. Phase 2: MVP 2.0 (The Alpha Build)**

While Assignment 3 was about *depth* (the single hardest feature), Assignment 4 is about *breadth* (the full loop).

**MVP 2.0 Requirements:**

1. **The "Game Loop":** The software must have a Start State (Menu), Active State (The experience), and End State (Win/Loss/Reset).  
2. **User Interface (UI):** Basic HUD, buttons, or controls must be functional.  
3. **Integration of Secondary Pillars:** If your second pillar was "Procedural Audio" or "Pathfinding," it must be integrated now.  
4. **Stability:** The build should run without crashing on standard input.

**Example Comparison:**

* *Assignment 3 (MVP 1):* A sphere floating in water with accurate buoyancy (Physics).  
* *Assignment 4 (MVP 2):* A boat that the user can steer through a procedurally generated ocean, with a start menu and a "Game Over" screen if it sinks.

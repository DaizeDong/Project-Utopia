## MVP 2.0

**1. The "Game Loop":**

- Start State: The game opens in a menu state and can start a run cleanly.
- Active State: The player can build, erase, inspect, pan, zoom, and manage a live colony on authored scenarios.
- End State: The build supports win/loss/reset flow instead of only an endless sandbox.

**2. User Interface (UI):**

- The alpha includes a functional HUD, Inspector, Entity Focus panel, AI decision panel, construction controls, and keyboard shortcuts.

**3. Integration of Secondary Pillars:**

- Already integrated (A* pathfinding, boids steering).

**4. Stability:**

- The local build runs, the test suite passes, and fallback browser long-run evidence exists for the authored scenarios.

## Release Notes

### New Features

- Added three authored scenario families with scenario-specific objectives, so each run begins as a real logistics repair problem instead of a mostly solved sandbox.
- Added the full alpha wrapper with menu, active play, win/loss/reset flow, HUD, Inspector, Entity Focus, AI panel, construction tools, and keyboard shortcuts.
- Added construction semantics, salvage, and build-preview feedback so map edits now have clear placement rules, costs, and scenario consequences.
- Added logistics, congestion, and route-pressure systems so warehouses, roads, and chokepoints visibly change throughput and A* routing behavior.
- Added doctrine, progression, and recovery systems so the colony now has explicit milestones and a limited comeback path instead of flat sandbox drift.
- Added visitors, trade, sabotage, animals, ecology, weather, and spatial events so pressure comes from concrete frontier locations instead of only global numbers.
- Added shared prompt payloads, policy-driven target selection, and common explanation surfaces so fallback and live-AI behavior stay structured and interpretable.
- Added deterministic soak, replay/snapshot, performance, long-run browser harness, and release-check tooling so the alpha can be validated as a product rather than only played manually.

### Bug Fixes

- Fixed active-state overlay input blocking and clarified the control scheme so the player can pan, build, select, and inspect reliably during play.
- Fixed worker state priority so AI steering can no longer trap core economy loops by overriding delivery and survival-critical local behavior.
- Fixed authored starts and logistics anchors that were mechanically invalid in some scenarios, especially around depots and warehouse tiles.
- Fixed predation accounting, wildlife recovery, and species telemetry so long-run ecology evidence reflects real deaths and does not silently lose herbivores forever.
- Fixed premature loss conditions and brittle long-run thresholds so runs are judged by real collapse instead of one-frame resource or depot-distance spikes.
- Fixed release verification gaps by adding stronger local build, artifact, and submission checks around the alpha build.

### Known Issues

- The full continuous `60m` fallback operator gate still needs a fresh rerun after the latest tuning pass.
- Live-LLM long-run validation is currently blocked by an upstream `401` API-key failure on this machine.
- Wildlife is now stable enough to avoid extinction in fallback idle runs, but herds still tend to settle at a low floor instead of a richer target population.

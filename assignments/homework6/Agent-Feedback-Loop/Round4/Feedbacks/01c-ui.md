# 01c-ui

## 总体评分

`3/10`

这是一个明显仍处于原型打磨期的 UI，不是可以按商业成品标准直接交付的界面。它并非完全不可用，首屏至少有明确标题、模式说明、模板选择、地图尺寸和开始按钮；进入游戏后也能看到资源、目标、建造工具、速度控制、帮助入口、资源与人口面板等核心系统。但问题在于：这些元素虽然“都在”，却没有被组织成一个成熟、稳定、低认知负担的信息系统。作为外部玩家，我能勉强理解玩法结构，却很难说这个 UI 已经形成了专业产品应有的主次秩序、视觉辨识度和操作反馈质量。

我基于当前浏览器构建进行盲审，只依据实际看到的标题页、帮助弹层、开局后的主 HUD、`Colony` 面板、`Build` 面板切换状态，以及我在截图时读取到的界面文本作出判断。以下结论不参考源码、文档、变更记录或任何前几轮信息。

## 第一印象

首屏给人的第一感觉是“信息很多，但不够会说话”。画面中央是一张深色半透明卡片，标题为 **Project Utopia**，副文案是“Reconnect the west lumber line, reclaim the east depot, then scale the colony.”，下面还有场景标签 **BROKEN FRONTIER · FRONTIER REPAIR · 96×72 TILES**。继续往下看，会立刻遇到生存目标条、模板下拉框、地图尺寸输入框、两排热键说明，以及 `Start Colony`、`How to Play`、`New Map` 三个按钮。

问题不在于缺内容，而在于全部内容几乎被摆在同一个层级上。标题不够“压住”场面，开始按钮虽然是高亮蓝色，但旁边两个按钮的存在感并没有被明显压低；模板与地图尺寸这种更像“高级设置”的内容，直接与主启动动作并列，打断了第一次进入时本该形成的清晰路径。背景 3D 地图非常暗，整体以深绿和深蓝为主，画面下方与两侧只隐约能看到地形、障碍和网格，玩家很难从首屏预览里获得“这个世界正在发生什么”的直觉。

换句话说，它不是没有 UI，而是 UI 还停留在“把系统摆出来”而不是“把体验讲清楚”的阶段。

## 信息架构

进入游戏后的信息架构问题比首屏更突出。当前我看到的主游戏 HUD 大致分成以下几块：

1. 左上角一列资源读数：`Food`、`Wood`、`Stone`、`Herbs`、`Workers`。
2. 顶部中间附近的场景目标条：`Broken Frontier — Reconnect the west lumber line, reclaim the east depot, then scale the colony.`
3. 目标条下面还有生存时间、分数、`Dev 49/100` 一类开发态/调试态文本，以及一组带有 `routes 1/1`、`depots 1/1`、`warehouses 4/2`、`farms 4/6`、`lumber 2/3`、`walls 7/8` 的任务状态。
4. 左侧是建造工具墙，从 `Select`、`Road` 到 `Clinic` 共十二项。
5. 左下/中下有建造说明与所选工具描述，例如 `Selected Tool Road`、`Cost: free`、`Rules:` 等。
6. 顶部右侧有 `Build`、`Colony`、`Heat Lens (L)`、`? Help` 四个按钮。
7. 底部中间附近有暂停、正常速度、快进和 `Autopilot`。
8. 打开 `Colony` 面板后，右侧又出现资源趋势、人口构成、工种数量等大量文字统计。

单看列表会觉得“功能很全”，但放在一个屏幕里就是典型的抢焦点。对一个陌生玩家来说，最重要的应该是三件事：我现在的目标是什么；我缺什么资源；下一步最推荐做什么。可当前界面把“目标进度”、“开发态指标”、“任务指标”、“里程碑文本”、“事件播报”、“建造清单”、“资源趋势”和“人口统计”都塞进同一轮可视范围，视觉层级没有拉开，导致主任务信息反而被噪声淹没。

尤其值得点名的是开发态文本的存在感过高。像 `Dev 49/100`、`Autopilot OFF - manual - coverage fallback` 这样的内容，在面向外部玩家的成品 UI 中应该被移到开发者面板、调试覆盖层，或默认彻底隐藏。现在它们直接出现在玩家视线中心附近，会严重损害专业感，也破坏世界沉浸。

## 视觉设计

优点是整体风格基本统一：深蓝半透明面板、蓝色高亮按钮、冷色系文本、轻微辉光和圆角，至少没有出现完全割裂的组件风格。标题页与游戏内 HUD 的美术语言是一致的，说明项目不是把默认浏览器控件胡乱拼起来，而是有过一轮统一皮肤处理。

但问题也非常明确：

1. **整体过暗。** 首屏背景的绿色地表几乎沉在黑里，地形预览价值有限。游戏内如果没有刻意盯着左上资源和左侧工具条，玩家第一眼很难判断场上单位、道路、建筑和空地之间的关系。
2. **文字太密、字号偏小。** 标题页两排控制说明、游戏内建造说明、`Colony` 面板中的资源趋势和人口统计都偏向“桌面调试工具”风格，不是易扫读的策略游戏 UI。
3. **图标语言薄弱。** 资源目前主要靠英文单词直接列出，工具主要靠文字按钮识别，缺少一眼可认的图标体系。相比成熟模拟经营游戏，这会明显拖慢信息读取速度。
4. **强调色使用不够克制。** 蓝色几乎承担了所有“这是互动元素”与“这是重点信息”的任务，但又没有更高一级的警报色、风险色或稀缺提示色形成有效分层，结果是界面很统一，却不够聪明。

从审美层面看，它像一个认真做过皮肤的内部测试版，而不是一套已经过大量可读性验证的成品 UI。

## 交互反馈

我实际确认到的正向反馈主要有三类：

1. 标题页按钮可点击，`How to Play` 会打开帮助弹层。
2. 游戏内 `Build` 与 `Colony` 按钮可以切换面板展示状态。
3. `? Help`、`Heat Lens (L)`、暂停/速度控制、建造工具栏都已经出现在界面上，至少说明操作入口是铺开的，而不是隐藏在多层菜单里。

帮助弹层是目前完成度相对较高的一块。打开后能看到明确分节：`Controls`、`Resource Chain`、`Threat & Prosperity`、`What makes Utopia different`，正文里还写了基础控制、开局建议以及 `ESC` / `F1` / `?` 重新打开帮助的说明。这部分的信息组织明显比主游戏 HUD 更像“给玩家看的内容”，也是我目前看到最接近成品表达的一处。

但从整体交互反馈质量来看，问题依然不少：

- 选中状态不够强。建造工具很多，但我没有看到那种非常明确的“当前工具已锁定”的高对比强调。
- 悬停反馈存在感有限。至少在可见层里，没有形成成熟的 tooltip 体系；有些描述更像浏览器 `title` 文本，而不是正式设计过的悬浮信息组件。
- 信息提示样式不统一。里程碑文本、状态播报、目标追踪和面板标题彼此都在竞争注意力，没有形成清晰的反馈优先级。
- 帮助、系统、调试、玩法提示混杂。对外部玩家来说，这会产生“哪里是游戏规则，哪里是开发备注”的困惑。

## 3D 呈现与 UI 协调

从当前证据看，3D 场景本身还不足以承担“主视觉”。标题页背景只给到一个很暗的地图预览：大面积深绿色地表、零散地块、重复性较高的纹理/标记、少量更亮的结构块。它能证明这是一个地图型策略游戏，但不能有效传达当前关卡的地貌特征、经营压力或视觉亮点。

进游戏后，UI 反而比 3D 场景更强势。左侧工具条、顶部目标、右上面板按钮、底部速度控制都在抢占空间，而场景本身没有足够高的辨识度来与之平衡。这会导致一种奇怪观感：界面很多，但世界不够“站得住”。成熟的殖民地模拟游戏通常会让玩家一眼认出地形、道路、建筑类别、危险区和单位行为；当前构建里，UI 在说“这是一个复杂系统”，但 3D 画面还没能把这个复杂系统可视化到同等水平。

如果用成品标准看，这种 UI/3D 关系是失衡的：不是 UI 太少，而是 UI 太多、场景太弱、两者缺少主次协调。

## 具体缺陷清单

以下缺陷均来自我已经捕获的实际界面状态：

1. **首屏主次不清。**  
   在标题页截图中，标题、任务描述、模式标签、目标条、模板下拉、尺寸输入、控制说明和主按钮全部集中在一个中等尺寸卡片里。视觉上没有“先看标题和开始，再看高级设置”的阅读路径。

2. **首屏背景预览价值低。**  
   同一张标题页截图里，背景地图过暗，边缘结构分布稀疏，除了知道这是地图类游戏，几乎读不出战略信息或地形个性。

3. **帮助信息比主界面更会说话，反衬主界面组织不足。**  
   `How to Play` 弹层中的分节结构、控制说明和开局建议很清楚，说明文案能力不是问题；问题是这些清晰表达没有反映到主 HUD 结构本身。

4. **开发态信息直接污染玩家视线。**  
   进入游戏后的截图里，`Dev 49/100`、`Autopilot OFF - manual - coverage fallback` 等字样直接出现在主视区。对于付费产品级 UI，这属于明显掉档次的问题。

5. **任务追踪写成了文本清单，而不是可快速扫描的状态面板。**  
   `routes 1/1`、`depots 1/1`、`warehouses 4/2`、`farms 4/6`、`lumber 2/3`、`walls 7/8` 这类信息很重要，但它们现在更像日志输出，不像经过图形化整理的目标追踪组件。

6. **左侧建造栏可用但笨重。**  
   十二个大号文字按钮能保证可点，但缺少图标、分类、优先级和视觉节奏，长时间使用会显得呆板且费扫读。

7. **`Colony` 面板信息密度过高。**  
   我看到的资源趋势包括 `▲ +82.7/min`、`▼ -57.4/min` 这类变化值，人口还细分到 `Visitors`、`Herbivores`、`Predators` 以及工种统计。问题不是数据不该有，而是面板几乎没有“概览层”和“细节层”的分工，第一次打开就被一整面数字扑脸。

8. **`Build` 面板切换后的空缺感暴露布局依赖。**  
   当我切换 `Build` 后，界面某些区域的内容密度发生明显变化，说明布局更多是在“把模块摆上去”，而不是围绕稳定的空间栅格规划。

## 响应式测试

按原要求，本应继续验证 `1920x1080`、`1024x768`、`800x600` 等尺寸下的可用性，并检查悬停 tooltip 与窗口缩放后的布局稳定性。但本次 Playwright 浏览器后端在若干次交互后出现反复断开，导致我最终只稳定完成了桌面大屏场景下的观察与截图，未能在这轮中完成可信的中低分辨率响应式验证。

因此，这一项我不给任何额外加分。以当前已经看到的信息密度和小字号倾向推断，中低分辨率下大概率只会更拥挤，不会更好。

## 与 RimWorld / Factorio / Oxygen Not Included 的 UI 对比

如果把这款作品放到这三类成熟作品的标准旁边，差距主要不在“功能数量”，而在“信息呈现成熟度”：

- **RimWorld** 的强项是状态优先级清楚。警报、殖民者状态、时间控制、底部工具栏和右侧面板彼此分工明确。  
  当前构建的问题是目标、资源、开发态文本和提示播报挤在一起，优先级没有被设计出来。

- **Factorio** 的强项是图标化与极快的扫读速度。  
  当前构建里资源、工具和状态过度依赖英文文字，阅读成本明显更高。

- **Oxygen Not Included** 的强项是复杂系统也能分层展示：概览给你决策入口，细节再逐层展开。  
  当前构建的 `Colony` 面板和主 HUD 更像把所有系统同时端上来，缺少折叠后的概览层。

如果只从 UI 呈现角度说，这款作品目前最需要学的不是“再加系统”，而是“学会隐藏系统，把真正该先看到的东西先给出来”。

## 改进建议

1. **重做首屏层级。**  
   让 `Start Colony` 成为绝对主按钮，把模板和地图尺寸收进“高级设置”或次级区块，热键说明缩成一行或折叠入口。

2. **清理开发态文本。**  
   所有 `Dev`、`coverage fallback`、调试流程字样都应默认隐藏，放入单独开发者面板。

3. **把任务追踪组件化。**  
   不要再用纯文字堆出 `warehouses 4/2` 这类状态，应该做成更短、更强对比、可一眼识别完成度的追踪模块。

4. **建立图标系统。**  
   资源、建造工具、警报和状态变化都需要图标化，至少让玩家在外围视觉里不用逐词阅读。

5. **降低同时在线的信息量。**  
   主 HUD 只保留最关键的资源、时间、目标和当前动作；趋势、人口细节、工种统计放进可展开二级面板。

6. **加强选中与悬停反馈。**  
   当前工具、可点击按钮、重要热点、不可用状态，都需要更明确的视觉差异，不应主要依赖文字说明。

7. **提高 3D 场景可读性。**  
   让道路、建筑、可交互地块和单位在远景下也能被区分，否则 UI 再努力，玩家仍会觉得“界面在说很多，但我看不懂地图”。

## 结论

当前版本的 UI 不是彻底失控，而是“原型功能齐、成品表达弱”。它已经具备一个策略/殖民模拟游戏所需的大部分入口，但尚未把这些入口整理成低负担、高辨识、可快速决策的视觉系统。帮助弹层说明团队知道如何把信息讲清楚，可惜主游戏界面还没有达到同样水准。按商业产品标准，我给 `3/10`：勉强能玩，远未精致，更谈不上成熟。

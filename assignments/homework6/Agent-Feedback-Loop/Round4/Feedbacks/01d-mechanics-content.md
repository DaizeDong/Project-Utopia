# 01d-mechanics-content

评审日期：2026-04-23  
评审对象：`http://127.0.0.1:4173/`

## 维度评分

- 机制呈现：`3/10`
- 内容丰富度：`2/10`
- 综合：`3/10`

## 总评

这是一个已经有“殖民地模拟器外观”的浏览器版本，但距离“机制被玩家真正感知到、内容被玩家真正消化掉”的状态还差得很远。它最明显的问题不是完全没做系统，而是**系统大多只以标签、数字、按钮、提示文案的形式存在**。玩家进入游戏后，确实会看到资源条、任务目标、里程碑、工具栏、速度控制、热力镜头、自动驾驶开关等一整套典型 colony sim UI；可一旦问一句“这些系统之间到底怎么彼此作用、玩家能不能稳定看到因果链、内容是否足够形成中长线策略差异”，当前构建给出的答案仍然偏弱。

我这次评审严格只依据当前浏览器构建，不读取源码、不读取项目说明、不参考过往轮次信息。基于已完成的实机观察，我的结论是：**UI 框架比机制体验成熟，目标文案比系统反馈成熟，建筑名录比实际内容厚度成熟。** 这会直接导致玩家第一眼觉得信息不少，但玩进来后很难确认哪些是已经成型的核心系统，哪些只是摆在那里的未来承诺。

## 机制呈现评分

我给 `3/10`。原因很直接：游戏有把“系统”摆出来，但没有把“系统运转过程”摆清楚。

开局我能立刻读到几类关键信息：Food、Wood、Stone、Herbs、Workers 五项资源/人口；场景标题 `Broken Frontier`；任务句子“Reconnect the west lumber line, reclaim the east depot, then scale the colony.”；生存计时、分数、`Dev 49/100`；若干进度块 `routes 1/1`、`depots 1/1`、`warehouses 3/2`、`farms 4/6`、`lumber 2/3`、`walls 7/8`；以及 `Next: Build Farm 4/6` 与 `Last: No deaths yet`。这些信息说明作者想做的不是纯铺地板，而是“任务驱动 + 资源链 + 生存/发展 + 建造规划”的复合玩法。

问题在于，这套机制更多还是被“写出来”，而不是被“演出来”。例如：

- 我知道有资源链，因为帮助界面有 `Resource Chain` 标题，且建筑名字暗示了 Farm/Lumber/Warehouse/Kitchen/Smithy/Clinic 的生产关系。
- 我知道有威胁与繁荣，因为帮助界面有 `Threat & Prosperity`。
- 我知道有 AI 或自动逻辑，因为界面显示 `Autopilot OFF - manual - coverage fallback`，还有 `Autopilot` 按钮。
- 我知道有热力镜头，因为界面有 `Heat Lens (L)`。
- 我知道有任务推进，因为出现了 `First extra Warehouse raised` 的里程碑提示。

但在当前可见体验里，这些系统大多没有被做成玩家能稳定观察、验证、归因的反馈链。玩家看到的是“有这个名词”，不太看到“这个名词在这一秒具体改变了什么”。资源是怎么被搬运的、路线修复具体如何影响效率、开发度如何计算、自动驾驶何时接管、热力镜头究竟揭示哪些瓶颈、仓库为什么会触发里程碑、里程碑对后续状态有什么持续影响，现有界面都没有给出足够强、足够持续的证据。

## 机制可视性清单

基于当前构建，我明确观察到的机制可视元素如下：

- 资源条：Food、Wood、Stone、Herbs、Workers。
- 场景与任务：Broken Frontier 场景说明，强调修复木材线、回收东侧 depot、扩张 colony。
- 发展指标：生存时间、Score、Dev。
- 目标计数：routes、depots、warehouses、farms、lumber、walls。
- 下一个引导目标：`Next: Build Farm 4/6`。
- 最近状态：`Last: No deaths yet`。
- 里程碑消息：`First extra Warehouse raised: The logistics net has a second anchor.`
- 速度控制：暂停、常速、四倍速。
- 自动功能：Autopilot 开关。
- 镜头/分析：Heat Lens (L)。
- 建造系统：左侧 Build/Construction 面板与 12 个快捷位工具。
- 选择/检查模式：Select，且帮助中明确写着可以点击 worker / visitor / animal 进行检查。

这份清单本身不短，但“可视元素多”不等于“机制呈现强”。当前问题在于可视元素彼此并没有形成强关联。玩家能看到很多控件，却不一定能从这些控件里读懂系统内部的压力传播。

## 因果链清晰度

因果链清晰度偏低。

最清楚的一条链，是非常基础的那条：打开 Build 面板，选择建筑，阅读右下/左下 Construction 说明，知道 Road “连接基础设施、减少搬运成本”，然后依据地图地块去放置。这条链至少有“工具 -> 规则文本 -> 地图对象”的闭环，虽然仍然偏浅。

除此之外，其他链条都显得发虚：

- 任务链：我知道要修复西侧木材线，但界面没有把“线路损坏点、修复前后差异、修复收益”做成特别强的前后对照。
- 仓储链：我知道额外建仓会触发 milestone，但不知道它对运输吞吐、工作半径、库存压力、后续发展究竟带来多大变化。
- 发展链：有 `Dev 49/100`，但玩家看不出 Dev 是由人口、建筑、产量、路径覆盖还是生存时间驱动。
- 自动化链：有 Autopilot，但从现有已见信息里难以判断它是在替玩家补路、自动补建筑、自动派工，还是只是展示状态。
- 生存链：有 `No deaths yet` 这类记录，但当前没有把死亡风险、伤病、敌袭、饥饿、天气灾害等压力系统稳定演给玩家看。

换句话说，游戏目前像是把 colony sim 的“名词表”搭起来了，但名词之间的箭头还不够粗。对于机制评测来说，这会严重拖分。

## 内容评分

我给 `2/10`。不是因为完全没有内容，而是因为**内容目前主要停留在工具名、标签名、系统预告名**，而不是玩家可持续消耗的真实内容层。

从 UI 表层看，似乎有不少东西：不同地图模板、不同建造项、不同模式按钮、不同状态条、不同说明页签。但真要从“可游玩内容密度”角度算账，当前实机能确认的内容远不够厚。

首先，地图模板虽有：

- Temperate Plains
- Rugged Highlands
- Archipelago Isles
- Coastal Ocean
- Fertile Riverlands
- Fortified Basin

但在我已完成的浏览器观察里，这些模板主要还是入口选择项，尚未证明每张图都会显著改变资源结构、地形难点、敌人压力、物流布局与中期策略。它们现在更像“可选种子包装”，不像已经建立鲜明玩法分叉的剧本集合。

其次，建筑列表虽然达到了十余项，可当前版本还没有把这些建筑背后的多层生产链、升级差异、维护成本、人员需求、季节/环境适配等内容深挖出来。只看当前可见状态，它们更像“一串可摆放名词”，而不是一个足以支撑几十小时推演的建筑生态。

再次，单位、事件、环境系统都偏少或偏弱。帮助界面提到 worker / visitor / animal，说明作者意识到实体种类应当不止工人；但在已见反馈中，除了工人与地图上少量移动小人/小动物图标外，没有建立起更丰富的社会层或威胁层。事件层面我目前只稳定看到里程碑文本，而非真正多样、能改变决策的事件池。环境层面目前清楚可见的是地块、水体、道路/桥梁适配与一个热力镜头按钮，还不足以撑起“环境系统丰富”这个评价。

- Wall

评价：**列表长度尚可，层次深度不足。**  
Road / Farm / Lumber / Warehouse 至少能让人读出最基本的物流与资源框架；Wall、Bridge 暗示防御与地形交互；Quarry、Herbs、Kitchen、Smithy、Clinic 暗示第二层生产和生存保障。但是在当前实机信息里，这些建筑之间的依赖关系、吞吐瓶颈、优先级冲突、空间布局博弈并没有被充分展开，所以还不能把“有这些建筑名字”直接算成“内容丰富”。

## 单位清单

当前能明确确认的单位/实体类型非常有限：

- Workers（资源栏明确显示 12）
- worker（帮助界面明确写到可点击 worker 检查）
- visitor（帮助界面提到）
- animal（帮助界面提到）

评价：这远远不够形成厚实的 colony sim 内容感。  
尤其是当一个游戏试图让人联想到 RimWorld 或 Dwarf Fortress 时，单位层的差距会被放大得非常明显。当前版本没有展示职业分工、需求差异、人格差异、装备层、伤病状态、迁徙访客行为、敌对派系、动物生态链等更丰富的实体内容。

## 事件清单

当前我能从实际构建里稳定确认到的事件/消息极少：

- `First extra Warehouse raised`
- `No deaths yet`

严格说，第二条更像状态记录，不像完整事件。  
这说明事件系统目前在玩家端几乎没有存在感。哪怕内部也许有更多逻辑，只要浏览器构建没把它们稳定演出来，对于评测就等于没有。

## 环境系统清单

当前能确认的环境/地图相关系统：

- 地图模板选择
- 地块尺寸预览
- 水域/桥梁适配
- 道路与已有基础设施连接规则
- 可缩放与拖拽地图
- 热力镜头按钮

评价：有基础，但还停留在“地图可操作”而不是“环境有性格”。  
我没有在已完成观察中看到足够强的天气、昼夜、季节、地形灾害、污染、温度、生物群落差异等环境反馈，因此不能给高分。

## 对比 RimWorld / Dwarf Fortress

如果拿 RimWorld 或 Dwarf Fortress 当标杆，这个版本在“内容量级”与“系统外显能力”上都差得很远。

RimWorld 的强项不是建筑名单长，而是每一种建筑、角色状态、事件、敌袭、疾病、交易、心情、地形、温度都在稳定地产生决策压力。Dwarf Fortress 更不用说，它的内容厚度来自世界、历史、实体、材料、工艺、事故、生态、社会结构的巨大叠加。

而当前版本还处在另一个阶段：  
它已经知道自己想成为“殖民地模拟”，也能把一些标准部件摆上屏幕，但这些部件之间还没有堆叠出那种让玩家持续惊讶、持续应对、持续复盘的内容体量。和这两部作品相比，现在更像是“带任务目标的可玩原型”，不是“内容型模拟器”。

## 改进建议

1. 先把一条资源链做透，不要只做宽。  
比如 Farm -> Warehouse -> Kitchen -> Workers 的完整链条，要让玩家能看到采集、运输、加工、消耗、断供、恢复的全过程。

2. 把 `Dev`、`Autopilot`、`Heat Lens` 这种高概念系统具体化。  
玩家必须能直接看懂这些按钮和数值改变了什么，不然它们只是在装饰复杂度。

3. 给任务做前后对照。  
“修复西侧木材线”不应只是一句任务文案，而应该让玩家明确看见断线、修复动作、吞吐变化、后续新建筑解锁或压力缓解。

4. 扩展事件池，而且事件要改变决策。  
不是多弹几条文本，而是让事件真正迫使玩家改建、调度、屯粮、防御或取舍。

5. 单位层必须加厚。  
工人、访客、动物至少要有更明确的状态差异和行为反馈，否则地图上的实体只是移动图标。

6. 地图模板要做出玩法分叉。  
不同模板应显著改变地形、资源获取方式、交通方案与风险来源，否则模板只是换皮。

## 结论

当前版本的优点，是已经搭出了一个“像样的 colony sim 控制台”；当前版本的致命问题，是**控制台比真正的系统更完整**。在机制呈现上，它还没有把玩家最需要看到的因果链条讲明白；在内容丰富度上，它也还没有超出“建筑名录 + 若干指标 + 少量里程碑”的原型期体量。基于当前浏览器构建，我认为这份作品在“机制/内容”这一维上仍然处于明显偏低分区间。

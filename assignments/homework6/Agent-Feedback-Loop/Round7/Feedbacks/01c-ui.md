# UI 评测报告 — Project Utopia v0.8.x
**评测日期：** 2026-04-26  
**评测身份：** 外部玩家盲审（首次接触）  
**评测角度：** 纯 UI 呈现——界面设计、信息层级、视觉清晰度、交互反馈、美术风格

---

## 总体评分：3.5 / 10

> 这是一款有内核的策略模拟游戏，但它的 UI 呈现质量远未达到"可发布产品"标准。信息架构混乱、视觉噪声严重、响应式完全失效，三者叠加使得初次体验非常痛苦。以付费产品标准衡量，当前 UI 无法过关。

---

## 第一印象

启动游戏后看到的是一个漂浮在黑色背景正中央的深色卡片式主菜单，左右两侧是大片无内容的黑色区域——菜单宽度约占屏幕的 55%，其余空间全部被背景中模糊的 3D 地形填充。主菜单没有全屏模式，没有居中拉伸，不管窗口是 1024px 还是 1920px，菜单卡片都固定在同一个位置，大量视觉空间被浪费。

标题字体"Project Utopia"使用了亮蓝色，视觉分量够，但与整体暗色卡片的对比过于突兀，缺少品牌感。副文本（场景描述）与 First Pressure 说明文字均为小号灰白色系，主次区分靠字号差异，但间距设计过密，导致整个主菜单卡片显得局促。

游戏进入后的第一眼：顶部 HUD、右侧边栏、左下角 Entity Focus 面板，三个区域同时争夺注意力，没有一个元素清晰地告诉玩家"你现在最需要关注的是什么"。这正是本 UI 最核心的问题。

---

## 信息架构

### 顶部 HUD

顶栏从左到右排列了 10 个信息元素：Food、Wood、Stone、Herbs、Workers（五个主要资源有图标+标签+数字+进度条），然后是 Meals、Tools、Medicine、Prosperity、Threat（五个次级数据，**仅有数字，无标签、无图标标注**）。

这意味着一个新玩家看到"21"和"39"两个数字时，完全不知道它们代表什么。经检查 DOM 发现这些数字分别是 Prosperity 和 Threat，但界面上没有任何文字线索。只有悬停 tooltip 才能得知含义——这对核心战略数据来说是完全不可接受的设计决策。

更严重的是：在 1280px 宽度下，HUD 右半区（Survived 时间、Autopilot 状态、场景标题）已经溢出屏幕右边界，被截断。玩家看不到自己存活了多久，也看不到当前的剧本目标。这是一级信息在默认分辨率下直接不可见。

### 右侧边栏

右侧边栏包含 Build / Colony / Settings / Heat / Terrain / Help 六个标签页，设计为垂直标签条。**关键问题：标签条宽度极窄（约 30px），文字被旋转 90 度竖排显示**。"Terrain (T)" 这样的标签名在侧栏上显示为竖排字母，辨识性极差。打开面板后，面板内容可读，但切换面板时的视觉焦点毫无引导——没有过渡动画，也没有选中态的边框或颜色区分（只有轻微的背景色变化）。

Build 面板里的建筑工具以 2 列网格排列，每个按钮有图标+文字，这部分设计相对清晰。但底部的 Construction 区域（显示所选工具的说明、造价、规则）字体极小（约 10-11px），在黑色背景上以灰白色显示，对比度勉强达标，长时间阅读非常吃力。

### 左下角 Entity Focus 面板

这是一个问题最集中的区域。面板高度约占屏幕下半部 40%，内容是一个可滚动的工人列表，每行格式为`姓名 · 职业 · 当前状态 · 饥饿程度`。字体约 12px，无色彩分级，hungry / starving / well-fed 全部用相同灰白色文字表示，没有视觉优先级。

面板底部是选中工人的简短背景说明，然后是一行绿色 toast 通知（如"First Kitchen raised"）。**这个 toast 与工人信息挤在同一行，二者逻辑上毫无关联，视觉上却紧贴在一起**，造成认知干扰。

更大的问题：在高速游戏（x4 加速）下，工人列表实时刷新，玩家根本无法稳定地阅读任何一行。这个面板的更新频率与其提供的信息密度完全失配。

### Colony 面板

Colony 面板包含 AI Storyteller 状态、殖民地状态（STABLE / Day 1）、详细资源列表（带每分钟变化率）、种群结构（Workers/Visitors/Herbivores/Predators + 各职业人数）。**这是整个游戏中信息最密集、最有价值的面板**，但它默认不显示——玩家需要先发现右侧边栏的"Colony"标签才能打开它。

相比之下，顶部 HUD 的精简版资源数字反而是在游戏中最醒目的位置。这种信息分层策略是合理的，但执行不到位：Colony 面板里的详细数据（如 Food: 202 ▼ -92.7/min，直到耗尽还有 1m 4s）比顶部 HUD 有用十倍，却被藏在了侧边栏里。

## 视觉设计

### 色彩

整体使用深蓝黑色背景（`rgba(12, 22, 35, 0.92)`），主文字颜色 `#c8e0f8`（淡蓝白），强调色 `#3d9be9`（明蓝）。配色方案本身有一定一致性，具有科幻策略游戏的气质。

但问题在于：

- **UI 层与 3D 世界的色调过于相近**。地图使用绿色/棕色/蓝色的瓷砖，UI 面板用深蓝黑色。当 Terrain 覆盖层（蓝色水域、深绿草地）铺满屏幕时，UI 元素几乎融入背景，边界模糊。
- **警报颜色不够醒目**。当 Wood 库存极低时，HUD 里的 Wood 进度条变为深红，但整体条状物极小（高度约 4px），在 12px 以下几乎看不出颜色变化。Threat 数值 39 以普通白色显示——没有任何颜色编码告诉玩家这是否危险。
- **Autopilot ON 的绿色醒目，Autopilot OFF 的黄绿色也挺醒目**——这一对状态处理相对合理，是少数做对的地方。

### 图标

工具栏图标使用了像素艺术风格 RPG 图标包（如 Apple.png 代表 Food，Wood Log.png 代表 Wood）。图标风格与游戏的 3D 顶视角审美有一定割裂感：游戏本体是等角 3D 瓷砖世界，UI 图标却是 2D 像素 RPG 风格。视觉语言不统一。

各个建筑按钮图标的辨识度参差不齐。Farm（向日葵图标）尚可，Smithy（锤子图标）也清晰，但 Quarry、Herbs、Clinic 的图标在按钮尺寸（约 20x20px）下难以分辨细节。

### 字体

全局使用 `"Segoe UI", "Trebuchet MS", sans-serif`，这是 Windows 系统默认字体栈——对一款游戏 UI 来说显得非常"网页化"，缺乏任何策略游戏应有的美术感。同时，字号层级混乱：

- 主 HUD 数字约 13-14px
- 工人列表约 12px  
- Build 面板说明文字约 10-11px
- Colony 面板资源变化率约 10px

最小的信息恰恰是最关键的（资源消耗速率），这与信息优先级完全相反。

### 地图 / 3D 场景

地图本身的地形着色尚可，草地/水域/泥土/建筑都有明确的色彩区分。但：

- 工人角色（白色小点）在浅色背景瓷砖上对比度很低，很难快速找到某个特定工人
- 地图节点标签（如"west frontier wilds"、"east ruined depot"）以黑底白字悬浮在地图上，字体大小约 11px，悬停时出现，这个设计合理但触发条件不明确——新玩家不知道要悬停哪里才能看到地名

## 交互反馈

### 按钮反馈

Build 面板的建筑选中状态使用深蓝色背景高亮（`--btn-active: rgba(61, 155, 233, 0.25)`），选中与未选中差异清晰。Speed 控制按钮（⏸ ▶ ⏩ ⏭）使用 Unicode 符号，功能明确，但按钮尺寸约 26x30px，偏小，在触控场景下会完全失效。

### Tooltip 系统

几乎所有 HUD 元素和按钮都有 `data-tip` 属性，提供了相当丰富的上下文信息。例如 Farm 的 tooltip："Farm (2) — produce food, cost: 5 wood. Place near soil-rich tiles; you'll need 3 before you can afford a Kitchen." 这些 tooltip 文案质量较高，信息量足够。

**但 tooltip 的触发没有视觉线索**——哪些元素有 tooltip，哪些没有，玩家无从得知。没有下划线提示，没有问号角标，没有光标变化。这需要玩家主动探索才能发现。

### Toast 通知

里程碑 toast（"First Kitchen raised"，"First Tool forged"）以醒目的绿色大字显示在地图正中央，**尺寸约占屏幕的 40% 宽度**，显示时间约 2-3 秒。这个设计选择非常激进——对核心视野造成大范围遮挡，而且尺寸远超信息量。在快速游戏节奏中，这样的全屏级通知干扰性极强。

底部左下角另有一行绿色小 toast 通知（如"Heat lens hidden"、"Auto-overlay: Overlay: Fertility"），与大尺寸 toast 形成双重通知系统，缺乏一致性。

### 键盘快捷键

快捷键方案丰富（Space 暂停、L 热力图、T 地形覆盖、1-12 建筑工具、Ctrl+Z/Y 撤销/重做），但：

- 快捷键仅在主菜单底部和 Build 面板底部的小字提示中出现，游戏进行中没有快捷键提示浮层
- Space 键在游戏内某些情况下会触发返回主菜单，这是一个严重的 UX bug——测评中多次意外触发

## 3D 场景呈现

游戏的 3D 等角视图本身构建得比较完善：瓷砖类型通过颜色区分，建筑物有简单但可辨认的 3D 形体，工人是移动的小白点，雨天效果（在 Island Relay 地图上可见雨滴效果）增加了场景动态感。

**然而 3D 场景与 UI 层缺乏视觉协调**：

- UI 面板使用半透明深色背景覆盖在 3D 场景上，但透明度不够——Entity Focus 面板的黑色背景遮住了左下角约 25% 的地图区域，玩家在看工人列表时看不到该区域发生的事情
- 地图 Overlay 标签（"Overlay: Fertility"）显示在左上角，字体 12px 白色，紧贴屏幕边缘，视觉权重过低
- 当 Terrain Overlay 激活时，地图整体变为蓝绿色调，此时右侧边栏的深蓝色 UI 与地图背景几乎同色，边栏与地图之间没有明确的视觉分割线

地图缩放后的可读性也有问题：缩小到可以看到整个 96x72 的地图时，工人点、建筑物图标都缩小到约 4-5px，完全无法辨认。游戏没有提供小地图（Minimap）替代方案。

## 响应式测试结果

| 分辨率 | 可用性 | 主要问题 |
|-------|-------|---------|
| 1920x1080 | 差 | 主菜单孤立漂浮，地图占用面积极小，侧边栏标签文字被截断 |
| 1280x800 | 勉强可用 | HUD 右半区溢出，侧边栏过窄，地图显示面积合理 |
| 1313x697（实测默认） | 基本可用 | 设计目标分辨率，大多数功能可用，但 HUD 仍有溢出 |
| 1024x768 | 不可用 | 侧边栏完全消失，HUD 严重截断，操作面板无法打开 |

**结论：游戏没有实现任何有效的响应式设计。** 它似乎针对一个约 1300x700 的固定分辨率设计，偏离这个分辨率就会出现严重的布局崩溃，且没有任何 breakpoint 处理或缩放自适应。

## 与同类游戏对比（RimWorld / Factorio）

### 信息层级

**RimWorld** 的做法：顶部状态栏只显示 5 个绝对核心数字（colonists/wealth/mood/food/etc），每个数字都有大图标和标签。细节信息放在点击开启的面板里，层级清晰。  
**Project Utopia** 的问题：把 10 个数字全部挤在顶栏，其中 5 个连标签都没有，信息层级崩塌。

**Factorio** 的做法：生产统计、地图概览、电网状态分属不同面板，玩家可以按需排列。操作中的工具提示精准到每个配方的产出速率。  
**Project Utopia** 的问题：Colony 面板的每分钟变化率数据（如 Food: -18.4/min）是很好的 Factorio 式信息，但字体 10px、对比度低，完全淹没在面板里。

### 警报系统

**RimWorld** 在右上角有一个独立的事件日志区，颜色编码（红色=紧急、黄色=注意），任何时候都能快速扫视。  
**Factorio** 在地图上显示工厂告警图标，配以声音反馈。  
**Project Utopia** 的 Entity Focus 面板底部有一行绿色 toast，但它同时承担了"工人选中状态"、"里程碑通知"、"叠加层状态"三种不同性质的信息，没有颜色分级，没有优先级排序，危急事件与普通通知混在一起。

### 可读性

**RimWorld** 在 1024x768 下完全可用。  
**Factorio** 在 1024x768 下 HUD 会略显紧凑但完全可操作。  
**Project Utopia** 在 1024x768 下侧边栏直接消失，核心操作面板无法打开。这是工程级别的失败，不是设计风格问题。

## 改进建议

**优先级 P0（必须修复，影响可用性）：**

1. **响应式布局：** 为 1024px、1280px、1920px 三个宽度设置 breakpoint。侧边栏应改为可折叠抽屉而非固定竖条，确保在任意分辨率下均可访问。
2. **HUD 标签补全：** Meals、Tools、Medicine、Prosperity、Threat 必须添加文字标签或图标，不能裸露数字。
3. **HUD 溢出修复：** Survived 时间和 Autopilot 状态应优先级高于中间的场景文字，在空间不足时场景文字截断而非关键状态截断。
4. **Space 键 bug：** 立即修复 Space 键在游戏中意外触发主菜单返回的问题。

**优先级 P1（显著影响体验）：**

5. **里程碑 toast 缩小：** 将全屏宽度的里程碑通知缩小为右侧滑入式通知条（参考 RimWorld 的事件日志），最大宽度不超过 300px。
6. **Minimap：** 添加可折叠的小地图，当缩放到全图视图时提供可读的区域概览。
7. **Tooltip 触发线索：** 为有 tooltip 的元素添加下划线虚线或角标问号，告知玩家可悬停查看更多信息。
8. **威胁警报颜色化：** Threat 数值超过 30 时变为橙色，超过 50 时变为红色闪烁；同样对 Prosperity 低值做绿色预警。
9. **Entity Focus 面板透明度：** 将面板背景透明度从约 0.9 降至约 0.75，或为其设置一个固定高度区域，避免遮挡地图主体。

**优先级 P2（提升质感）：**

10. **字体升级：** 引入一款具有策略游戏气质的 Google Font（如 Rajdhani、Exo 2），替换系统默认字体。
11. **字号层级重组：** 将资源变化率（如 -18.4/min）字号提升至 12px，增加对比度；将工人列表字号从 12px 降至 11px，换取更多可见工人数量。
12. **图标风格统一：** 要么将 HUD 图标换为与 3D 场景风格一致的等角图标，要么将 3D 建筑物风格向像素艺术靠拢，消除视觉语言割裂。
13. **主菜单背景利用：** 将主菜单背景的 3D 地形渲染改为实际的地图预览动画，或至少让主菜单卡片居中拉伸以填充屏幕宽度的 70-80%。

---

## 总结

Project Utopia 的 UI 存在三个核心性问题：**信息层级混乱**（核心数据没有标签，次要信息与关键信息视觉权重相同）、**响应式完全缺失**（1024px 即崩溃，1920px 极度浪费）、**通知系统设计失控**（里程碑 toast 过于巨大，多套通知并行且无优先级）。这些问题不是"打磨不足"，而是架构层面的设计缺陷，需要系统性重构而非小修补丁。游戏的内容深度和 tooltip 文案质量值得肯定，但 UI 这层皮远未到位。

**3.5 / 10** — UI 呈现未达可发布标准，响应式测试直接不及格，信息架构需要重新设计。

# Project Utopia — 可玩性外部评测报告

**评测版本**：v0.8.x（本地开发构建，http://127.0.0.1:5173/）
**评测日期**：2026-04-26
**评测角色**：苛刻的外部玩家评测者（盲审，不查阅源代码）
**总体评分**：**3.5 / 10**

---

## 总体评分速览

| 维度 | 评分 | 简评 |
|------|------|------|
| 新手引导 | 4/10 | 有文字说明但缺乏互动教学 |
| 前30分钟体验 | 3/10 | 饥荒压力极端，玩家几乎无法介入 |
| 中期深度 | 4/10 | 资源链设计合理但呈现晦涩 |
| 长期耐玩性 | 3/10 | 无可见的长期目标弧 |
| 决策深度 | 3/10 | 建筑放置是唯一真实决策 |
| 反馈循环 | 4/10 | 数字详细但情绪感染力弱 |
| 操控手感 | 2/10 | 画布点击不可靠，面板切换混乱 |
| 情绪曲线 | 3/10 | 焦虑持续但无法形成满足感 |

**总分：3.5 / 10**

---

## 第一部分：前30分钟体验

### 进入游戏

主菜单界面整洁，提供6个地图模板（Temperate Plains、Rugged Highlands、Archipelago Isles、Coastal Ocean、Fertile Riverlands、Fortified Basin），每个模板配有简短的"场景简报"描述首要压力和建议首步行动。这部分做得不错——至少让玩家知道"要做什么"。

然而问题立刻出现：**玩家没有真正的新手教学关卡**。"How to Play"按钮打开一个模态框，里面是纯文字描述资源链和快捷键。对于一个系统复杂度超过Banished的游戏，这种"读说明书"式教学是完全不够的。

### 开局即饥荒

殖民地一旦启动，几乎所有工人（12人）都处于"hungry"（饥饿）或"starving"（濒死）状态。资源面板显示Food消耗速率约为 **-62.5/min**，而倒计时显示"约48秒后粮食耗尽"。

这种开局设计是毁灭性的。玩家还没搞清楚鼠标怎么点、建筑怎么放，工人就已经开始大规模挨饿。从情绪层面来看，新玩家的第一感受将是**纯粹的恐慌**，而非策略性的兴奋。

更糟糕的是，这种恐慌感无法通过玩家行动有效消解。建筑放置需要找到正确的地块，而大量区域是"未探索地形"，点击它们只会看到"Cannot build on unexplored terrain. Scout this area first."这条信息连续出现六次，玩家一边找地方建农场，工人一边继续挨饿，这是设计上的残忍，而非刺激。

### 操控体验极差

游戏的核心交互是在Three.js渲染的画布上点击放置建筑。这个机制在实际测试中极度不可靠：

- **画布点击触发页面刷新**：在Farm工具激活状态下点击画布区域，多次导致页面完全重新加载（console错误"Execution context was destroyed"）。这直接导致数分钟的游戏进度清零。
- **帮助模态框阻塞全局操作**：Modal出现时阻止所有背景点击，ESC键关闭不够直觉，玩家可能找不到关闭路径
- **Build标签切换副作用**：点击"Build"标签有时触发全局暂停（"Simulation paused"），不清楚是否为预期设计
- **建筑工具面板可见性**：工具按钮（Farm、Road等）只在特定面板状态下可见，切换到Colony或Settings后消失，容易让玩家迷失

### 随机场景跳转

测试过程中，游戏在以下场景之间随机切换：Temperate Plains / Broken Frontier → Fortified Basin / Hollow Keep → Rugged Highlands / Gate Bastion → Archipelago Isles / Island Relay → Fertile Riverlands / Silted Hearth。

每次切换发生时，玩家正在建立的心智模型被彻底清空。虽然部分切换可能由操作触发，但这种不稳定性本身就是可玩性问题：一款好游戏不该让玩家不断质疑"我现在到底在哪张地图上"。

## 第二部分：中期体验（Day 3–10）

### AI代劳的殖民地

开启Autopilot（AI自动驾驶）后，游戏在约4分35秒内将殖民地推进到Day 5，此时：
- 23名以上工人全部处于"well-fed"（饱食）状态
- 威胁等级升至40%
- Smithy（锻造厂）正在运转
- DIRECTOR AI显示"picks rebuild the broken supply lane"

这里暴露了一个根本性的设计矛盾：**AI自动驾驶比人类玩家更能有效管理殖民地**。手动模式下，殖民地通常在1-2分钟内因粮食耗尽而崩溃；开启Autopilot后，殖民地到Day 5依然欣欣向荣，全员饱食。这个对比太鲜明，令人尴尬。

当游戏内置的AI比玩家做得更好，而且玩家的主要操控手段（建筑放置）在操作层面又如此不可靠，玩家的参与感就会从"我在掌舵"滑向"我在看直播"。

### 资源链：好想法，差呈现

游戏设计了一套处理链：
```
Food(农场) + Kitchen → Meals（饥饿恢复×2）
Wood(伐木场) → 主要建材
Stone + Wood → Smithy → Tools（+15%收获速度）
Herbs(草药园) + Clinic → Medicine（治疗伤病）
```

这套设计有合理的策略深度：玩家需要权衡先建Kitchen还是先扩张农场，Tools的加速效果是否值得石料投入。然而这些权衡从未被清晰地呈现给玩家。资源面板的数字详细（带变化率和ETA），但没有一个直观的"当前生产链健康状况"视图。

"input starved"（输入匮乏）的警告频繁出现，但它没有告诉玩家：哪个建筑在缺乏输入？缺什么？建什么能解决？这是信息有余、指引不足的典型问题。

### 工人系统：技术炫耀多于玩法服务

每个工人都有完整的人格档案：姓名、职业背景、性格特征、心情、士气、社交值、休息值、详细人际关系（Friend/Close friend）、近期记忆列表，以及"Why is this worker doing this?"的决策透明度。

一个工人的决策理由样例：
> "Carry's still light — I'll keep working the farm site until my hands are full. Current route crosses congestion near (48, 32) with lane load 3.0 and x1.24 path cost."

这些技术细节令人惊叹，但它们的存在方式是只读展示，玩家无法基于这些信息采取任何干预操作。工人完全自主，玩家只能观察。在无法干预的情况下，再丰富的角色信息都只是装饰品。

## 第三部分：长期耐玩性

### 没有叙事弧，只有等待崩溃

游戏模式是"无尽生存"（Survive as long as you can）。全程测试没有看到明确的中期里程碑：
- 系统自动弹出toast通知（"First Kitchen raised"、"First Meal served"）算是小里程碑，但感觉更像系统日志而非奖励时刻
- Best Runs记录板全程显示"No runs yet"，因为殖民地没能撑到被记录的地步
- 无尽生存缺乏弧度——玩家不知道"我要活到Day X才算成功"

### 威胁系统：存在感极弱

威胁指示器从22%逐步上升到40%，场景简报提到"Raiders"（突袭者）和"hold north and south before raiders find the breach"，但测试过程中没有看到任何实质性的防御交互。威胁只是一个百分比数字，没有配套的视觉冲突、音效警报或"敌人快来了"的前兆。

这是一个严重的设计缺口：游戏告诉玩家要防御，但既不展示威胁形态，也不提供直观的防御反馈，威胁系统对可玩性的贡献接近零。

## 第四部分：决策深度分析

### 真正的决策点在哪里？

玩家唯一真实的决策权是**建筑放置**。道路、农场、仓库、厨房、锻造厂的位置直接影响物流效率——这是游戏里最接近策略的部分。

然而这个决策点受到多重限制：
1. **视野限制**：大量区域是"未探索地形"，无法建造
2. **操控不可靠**：画布点击机制有严重Bug
3. **即时反馈缺失**：放置建筑后不清楚工人何时会响应
4. **建造成本模糊**：农场需5木材，厨房需8木材+3石料，但玩家没有足够时间理解这些成本如何影响长期战略

还有"Target Farmer Ratio: 50%"的滑块可以调整，但没有说明调整对局势的即时影响，玩家拉了滑块，不知道发生了什么。

### 与经典同类游戏的对比

| 对比维度 | RimWorld | Banished | Frostpunk | Project Utopia |
|----------|----------|----------|-----------|----------------|
| 玩家控制粒度 | 精细到个人指令 | 区域性建筑决策 | 资源+法令系统 | 仅建筑放置+滑块 |
| 开局压力 | 渐进式，有喘息时间 | 温和，有充裕学习期 | 极端但有明确指引 | 极端，无指引 |
| 失败可读性 | 高（死亡日志详细） | 高（资源趋势清晰） | 高（倒计时、警报） | 低（崩溃突然） |
| 新手友好度 | 一般（依赖wiki） | 良好 | 一般（有叙事引导） | 差 |

**RimWorld** 给玩家直接分配工作的权力，工人只有在没有指令时才自主行动。Project Utopia完全相反——工人完全自主，玩家仅通过建设间接影响他们。这种设计理念本身有趣，但"间接控制"必须有足够清晰的因果可见性，否则玩家感觉不到自己的存在意义。

**Frostpunk** 的极端压力之所以成立，是因为每个法令决定都立即可见地影响城市状态，温度计、满意度条、工人血量——所有变量都在一个屏幕上直观呈现。Project Utopia的极端压力没有对应的清晰控制权。

**Banished** 看似简单，但它的核心循环高度清晰：建房子→工人搬来→资源产出→人口增长，每个建筑效果5分钟内肉眼可见。Project Utopia的资源链更复杂，但效果更难感知。

## 第五部分：反馈循环分析

### 短期反馈（0-2分钟）

**存在的**：
- Toast通知（"First Kitchen raised"、"First Meal served"）
- 资源数字实时变动，带变化率（+42/min）和ETA
- 工人状态标签（hungry / well-fed / starving）实时更新

**缺失的**：
- 建筑放置成功的视觉/音效确认
- 工人响应新建筑的明显动画
- 建造操作与效果之间的清晰因果展示

### 中期反馈（2-10分钟）

**存在的**：
- 工人数量通过出生事件增长（12→23+）
- Dev指数（Dev 40-44）存在，但玩家不清楚它的含义和意义

**缺失的**：
- 明确的"你在进步"信号
- 建设成就的视觉庆典时刻
- "生存X天"之类的里程碑感

### 长期反馈（10分钟+）

几乎不存在。无尽生存模式没有弧度，威胁升级缺少戏剧性事件，殖民地历史没有积累感。

## 第六部分：情绪曲线分析

理想的殖民地模拟游戏情绪曲线：
```
焦虑 → 努力 → 微小胜利 → 满足感 → 新压力 → 循环
```

Project Utopia实际曲线：
```
困惑（找不到建造位置）→ 恐慌（工人饥饿倒计时）→ 混乱（操控失灵/页面刷新）→ 无力感（崩溃）→ 困惑（回到主菜单，换了地图）
```

测试过程中的主要情绪是**挫败**而非挑战带来的刺激感。每次殖民地崩溃不是因为做出了错误的策略决定，而是因为：
1. 操控系统不可靠，无法稳定放置建筑
2. 时间压力极端，容不得任何学习曲线
3. 不知道第一步该做什么、做对了有什么回应

唯一的正向情绪出现在开启Autopilot后看到殖民地顺利运转——这是对游戏可玩性最辛辣的讽刺。

## 第七部分：记录到的Bug与技术问题

1. **画布点击触发导航刷新**：Farm工具激活时点击画布，多次导致页面完全重新加载（console: "Execution context was destroyed"），游戏进度清零
2. **帮助模态框无法通过ESC关闭**：需要手动调用JS隐藏才能恢复正常操作
3. **"Cannot build on unexplored terrain"重复提示**：同一条错误信息连续出现6次以上，没有聚合处理
4. **速度显示不一致**：实际速度在×2.5、×3.7、×4.0、×5.8之间跳动，与UI标注的固定倍速不符
5. **Build标签触发意外暂停**：点击Build标签有时触发"Simulation paused"，行为不符合玩家预期
6. **资源正负号矛盾**：食物显示"+42/min"但ETA显示"2m 19s until empty"，实际消耗超过产出，正号显示误导玩家
7. **AI Log面板7个JavaScript errors**：打开AI Log时console报错，功能稳定性存疑

## 第八部分：改进建议（按优先级排序）

### 高优先级（影响核心可玩性）

**1. 修复画布点击可靠性**
建筑放置是游戏唯一的核心操控。画布点击触发页面刷新是致命缺陷，必须首先解决。建议增加click-on-canvas的调试日志，找出导致导航的根本原因。

**2. 降低开局食物压力**
给新玩家至少3-5分钟的初始宽松期。建议开局储备食物增加至500+，消耗速率降低50%，让玩家有时间熟悉UI和建造流程再面对真实压力。

**3. 添加互动式新手教学**
第一局必须是手把手引导：高亮"在这里建农场"→工人移动的动画→第一次食物收获→系统解释why。纯文字模态框无法代替操作引导。

**4. 明确第一个操作的位置**
场景简报说"Build a road to the west forest"，但地图上没有标注"west forest"在哪里。第一局开始时，地图上应该有箭头或高亮区域告诉玩家点哪里。

### 中优先级（影响中期体验）

**5. 建立中期里程碑目标阶梯**
增加明确的阶段性目标："存活10天"、"建立完整食物链"、"人口达到20人"。每个里程碑配有小型视觉庆典，给玩家阶段性满足感。

**6. 简化工人信息面板的默认视图**
保留核心信息（饥饿状态、当前任务、健康值），把详细心理数据（情绪值、关系树、决策推理）折叠到"展开详情"区域。信息分层非常重要。

**7. 提升建筑效果的直觉可见性**
放置新建筑后，附近工人应该有明显的"被吸引"动画，工人数量改变时要有清晰的+/- 指示符，让玩家立即感知决策效果。

**8. 解决资源正负号混淆问题**
食物净消耗时显示负数（-40/min），净产出时显示正数（+42/min），保持一致性，避免误导。

### 低优先级（改善整体体验）

**9. 加入"轻松"难度选项**
为新玩家提供2-3倍的初始资源、更低的消耗速率，确保他们能在不崩溃的情况下探索所有系统。

**10. 为工人添加职业色彩编码**
当前地图上的工人小人几乎无法区分。用颜色编码区分FARM/WOOD/STONE等职业，帮助玩家快速感知劳动力分布。

**11. 威胁系统的前兆与视觉化**
在敌人到达前添加预警阶段（地图边缘出现敌方单位的影子？），让"Threat: 40%"不只是一个抽象数字，而是有感知的危险信号。

---

## 最终总结

Project Utopia是一款**技术野心远超可玩性执行**的殖民地模拟游戏。工人AI系统的深度（性格、关系网络、决策透明度）在独立游戏中属于顶级水准，资源链的设计思路有合理的策略潜力，6种地图模板带来了有差异化的初始场景。

然而，这些技术成就没有转化为有趣的玩家体验。操控系统不可靠（建筑放置Bug）、开局压力极端（12人即饥荒）、玩家实际影响力有限（AI驾驶比人类更有效）、反馈循环薄弱（数字详细但情感联结弱）——这四个核心问题共同制造了"看别人玩比自己玩更好"的奇异处境，而"别人"就是游戏自带的Autopilot。

**最终评分：3.5 / 10**

> "一个充满想法的模拟器，但尚未成为一款真正可以玩的游戏。当AI托管表现优于玩家手动操控，这款游戏的可玩性设计需要从根基开始重新审视。"

*评测说明：通过浏览器自动化工具进行60+次交互操作（点击、键盘快捷键、面板观察），观察了Temperate Plains、Fortified Basin、Rugged Highlands、Archipelago Isles、Fertile Riverlands等多个地图的早期至中期游戏状态（最长观察至Day 5, 4分35秒游戏时长，开启Autopilot）。全程未查阅源代码。*

# 01b-playability 盲测反馈

日期：2026-04-26  
测试角度：只看 moment-to-moment playability，不评价源码、架构或作者意图。  
证据截图目录：`screenshots/01b-playability/`

## 总评分（10 分制）

**5.5 / 10**

Project Utopia 已经有一个能跑起来、能观察到资源变化、工人分工、事件压力和 AI/autopilot 介入的游戏循环雏形。它不是空壳：我能看到食物、木材、石头、工人、路线节点、仓库、厨房、餐食、天气/商队事件和工人职业状态在变。但从付费玩家的“我点了什么、为什么点、点完发生了什么、后果是不是我造成的”角度看，当前可玩性仍然卡在“系统很多、反馈很多、但决策感不稳定”的阶段。它更像一个复杂模拟器仪表盘，而不是一个已经把玩家手感打磨清楚的策略生存游戏。

## 第一印象

开局菜单给的信息很明确：西侧伐木路线要恢复，东侧废弃仓库要重建。这个目标比普通沙盒开局好，因为玩家一进来就知道“先修路、再仓库”。进入游戏后，地图视觉也能看出主岛、东南岛、蓝色水域/阻断、以及几个带标签的关键点，例如 `west lumber route`、`west frontier wilds`、`east ruined depot x2`。这一点对可玩性有帮助。

问题是，真正开始操作后，界面同时给出太多层信息：顶部资源、顶部叙事条、底部速度条、右侧工具栏、左下工人列表、浮动提示、事件文本、覆盖层标签、AI 状态。它们都在“解释游戏”，但玩家很难判断当前最重要的下一步是什么。第一次手动尝试道路、仓库、农田、伐木、采石、草药时，我确实看到资源数字和工具说明变化，但很少有强烈的“我刚刚解决了一个问题”的瞬间。

## 核心可玩性断点

最大的断点是**手动建造缺乏足够强的确认反馈**。我连续在西侧路线附近点了多段 road，又尝试在东侧废弃平台附近建 warehouse。地图上似乎有道路/建筑被放下，木材从 27 快速掉到 4、石头掉到 1，说明系统确实扣了资源。但对玩家来说，关键不是“资源扣了”，而是“西侧路线是否因此恢复”“东侧 depot 是否真的被 reclaimed”“工人是否会因此更高效”。这些后果没有以足够清晰、即时、可归因的方式出现。

第二个断点是**等待和快进时，变化很多但因果不清**。我在手动开局后切到 normal/fast，能看到工人从 Idle 变成 Seek Task，也看到出生提示、traffic hotspot、资源速率和事件文本。但我很难判断这是我刚刚修路/建造导致的，还是系统脚本自然推进导致的。快进时资源波动很大，autopilot 下木材曾显示约 `-1013/min`，随后又变成正增长，但没有足够好的解释告诉我“这是一次性建造消耗、供应链堵塞、天气影响，还是 AI 调度策略”。

第三个断点是**失败和危机可见，但不够可理解**。Dova 一开始就 starving，之后新生儿也可能 starving；storm、trade caravan、contested zones、pressure、path cost 等信息出现得很早。它们制造了紧张感，但没有转化成清楚的玩家任务。玩家知道“有危险”，却不知道最有效的处理动作是补农田、补厨房、修路、开 autopilot、还是只等工人搬运。

## 具体场景问题列表

- 手动道路：在西侧 `west lumber route` 附近连续铺路后，视觉上能看到地图有变化，但目标进度没有用强反馈告诉我“0/1 route online”是否变成“1/1”。如果玩家没有盯右侧 debug/警告文本，很容易不知道自己是否做对。

- 东侧仓库：我按开局提示尝试在 `east ruined depot x2` 附近建 warehouse。建造后木材/石头明显下降，但东侧岛仍然看起来隔着水，是否完成“rebuild east warehouse”目标不直观。这个开局任务应该有更强的完成动画、音效或 checklist。

- 草药建筑：选择 Herbs 后，右侧明确提示 “No herb node on this tile”，这条负反馈是清楚的，是本轮测试中较好的无效操作反馈。但它只解释“不能建”，没有引导我到哪里能建，导致下一步仍靠猜。

- 工人列表：工人名字、职业、Seek Task、hungry/starving 信息很有生活感，但作为操作对象偏弱。我点击/尝试选择工人时，并没有形成“我能命令这个人解决问题”的感觉，更像只是在看一个日志列表。

- 事件反馈：`First Kitchen raised`、`First Meal served` 是很好的里程碑提示，我看到它们时会觉得殖民地真的在推进。但这些提示在手动与 autopilot 下都可能发生，因此它们没有充分证明玩家操作有效。

- 速度控制：pause、normal、4x、8x 的存在很必要。问题是当前快进会把信息洪水放大，玩家还没理解当前供应链，就已经看到出生、天气、商队、压力、资源速率同时变化。

- Heat Lens / Terrain Overlay：覆盖层有潜力帮助玩家理解地图，但实际切换后，红/蓝/肥力/连通性的含义仍要靠文字解释。对普通玩家来说，颜色变化没有直接转化为“我应该点这里”。

- Autopilot：开启后很快能看到 AI 决策文本、厨房、餐食、路线处理和事件应对。它明显比手动更“会玩”，但这也削弱了手动学习的价值。

## 操作 / 反馈 / 后果链评价

当前链条大致是：玩家选择工具，点击地图，资源扣除，地图出现小变化，工人状态变成 Seek Task，若等待足够久，系统产生里程碑或事件。这条链是存在的，但每一环之间的连接不够紧。

好的部分是工具说明较具体，例如 Road 说明它能减少 hauling，Herbs 会说明需要 herb patch，Warehouse/Farm/Lumber 等工具都能让玩家预期资源链。资源 HUD 也足够醒目，能看到食物、木材、石头、工人数量变化。autopilot 的 AI decision 文本也能把系统意图暴露出来，例如 workers 要 rebuild broken supply lane。

弱的部分是后果没有被绑定到玩家动作上。比如我建了若干道路后，系统没有弹出“west lumber route restored: wood workers now save X path cost”。我建造厨房/仓库相关设施后，出现了 `First Meal served`，但没有告诉我是哪个建筑、哪个工人、哪条路线让它发生。资源速率有数字，但缺少可解释拆分；`Food +267/min` 和 `Wood -1013/min` 对策略判断太粗暴，像调试指标多于玩家反馈。

## dominant strategy 或无效选择

目前最明显的 dominant strategy 是**开 autopilot，然后快进**。对照测试里，autopilot 启用后能自动形成厨房、餐食、工人分配、AI policy、事件应对，玩家几乎不用理解地图点击规则就能看到殖民地变“活”。手动建造反而更容易陷入资源被扣、目标没确认、工人忙但不知为何的状态。

手动选择里，Road 看起来是必选项，因为开局目标直接要求恢复路线，且道路免费；Warehouse/Kitchen 也像强制路线，因为里程碑反馈明显。Farm、Lumber、Quarry、Herbs 的选择价值相对不清，尤其在资源节点和合法地块不明显时，玩家容易通过乱点试错。Erase、Bridge、Wall、Smithy、Clinic 在早期没有被情境化，像是“可能有用但现在别碰”的按钮。

## 改进建议

- 给开局任务做明确 checklist：`修通西侧伐木路线 0/1`、`重建东侧仓库 0/1`、`保持食物收入为正`。每次点击后如果相关进度变化，直接在地图和 HUD 同时确认。

- 把建造反馈做成三段式：点击前显示合法性和预期收益，点击后显示“已排程/已建成/缺资源”，完成后显示实际后果，例如路线恢复、搬运距离下降、某工人开始执行。

- 快进时降低信息噪音。4x/8x 下可以把出生、天气、商队、资源速率、AI 决策按优先级折叠，只保留真正需要玩家响应的危机。

- 让 worker 行动更可读。点击工人后应高亮他的目标建筑、路径、当前需求和失败原因，例如 “去伐木点，缺路，预计 12s”。这样工人不只是 flavor 文本。

- Autopilot 不应只是替玩家玩。建议让它输出可执行建议并等待确认，或者把它的动作标注为“我为什么这样建”，让玩家学习后能手动复现。

- 对无效建造给出下一步引导。`No herb node on this tile` 后可以高亮最近 herb node；桥不能建时提示需要连接哪两岸；仓库不适合时提示哪个废墟平台有效。

- 把资源危机解释成人话。不要只显示 `pressure 0.67`、`peak x1.72 path cost`，应显示“东侧仓库未接通，搬运绕路，木材供应会在约 N 秒后停滞”。

## 一句话总结

Project Utopia 已经有可观察的生存模拟循环，但手动操作的即时反馈和因果确认不足，导致当前最有效也最省心的玩法是开 autopilot 快进，而不是亲手规划并理解殖民地。

# Project Utopia —— UI 呈现外部评测

**评测日期：** 2026-04-22
**评测人身份：** 首次接触本作、面向成品商业游戏标准的付费玩家
**评测维度：** 仅限 UI 呈现（界面设计、信息层级、视觉清晰度、交互反馈、美术风格）
**评测方式：** 本地 `http://127.0.0.1:5173/` 浏览器交互，分辨率多档测试（1920×1080 / 1024×768 / 800×600 / 600×900），全程未读源代码

---

## 总体评分：**3.5 / 10**

一句话总结：**一个明显"工程师自用调试台"气质极重的模拟器原型，信息全但毫无美术把控，在成品商业游戏的标尺下属于"勉强能玩、完全不能卖"的状态。**

---

## 一、第一印象（0～30 秒）

打开页面后，扑面而来的是一个 **深蓝灰主题的浮层开始面板**，标题 "Project Utopia" 使用了较大的蓝色字体，子标题写着"Reconnect the west lumber line, reclaim the east depot, then scale the colony." ——这是个任务提示，但在一个**首次进入的玩家**眼里完全没有前因后果（谁毁了西部伐木线？"东仓"是什么？）。下面挤着三条标签：

- `BROKEN FRONTIER · FRONTIER REPAIR · 96×72 · SEED 1337` —— 用大写蓝字徽章式呈现，像是开发者调试元数据而不是玩家文案。
- 一张目标卡 `Survive as long as you can  00:00:00 · 0 pts`；
- 紧接着 `Template` 下拉、`Map Size 96 × 72 tiles` 输入框；
- 快捷键提示行 `RMB drag to preview · Scroll zoom · 1-12 tools · Space pause · L heat lens`；
- `Start Colony` / `New Map` 两个按钮，`Start Colony` 带一个冒出的小 tooltip "Begin the simulation with this map"。

**观感**：开场就像 Vite 启动后的一个 React devtool 弹窗，完全不是 2026 年的商业 sim 游戏首屏。背景里确实有已经在跑的 3D/2.5D 场景（绿地、蓝色河流），但没有任何角色剧照、logo、场景动画、音乐或摄影机切镜的引导。若对比《Oxygen Not Included》或《Frostpunk》的开场，差距一望而知。

进入游戏后（Start Colony 后），**开始面板消失，HUD 一次性全部怼到脸上**，没有任何渐显或新手引导脉冲，零过渡。

## 二、信息架构 / 信息层级

### 2.1 顶栏资源条（最严重的 UI 问题之一）

顶栏极扁（大约 24px 高），贴在屏幕最上缘，**一字排开挤着 10 个图标 + 数字**：
Food / Wood / Stone / Herbs / Workers / Meals / Tools / Medicine / Prosperity / Threat。
紧跟着一条 `Survived 00:00:01 Score 0 · Dev 50/100` 小字说明，再后面跟一条滚动的黄色/红色 banner `Simulation started. Build...` / `Emergency relief stabilizer` / `Insufficient resources` / `Cannot build on water tile` / `No forest node on this ti...`。
最右是五个 panel toggle：`Build | Colony | Settings | Debug | Heat Lens (L)`。

问题集群：

1. **图标太小**：每个资源图标目测 ~16–18px，数字更小（约 12–13px 的浅灰 sans-serif）。在 1920×1080 下已经难辨识，1080p 显示器离屏幕 70cm 基本看不清 `8/9/0` 之间的区别。
2. **无颜色编码**：所有资源数字用同一个灰白色，只在图标下方有极窄的进度条（约 2px），颜色仅有绿/橙区分；`Food 100` 和 `Food 8` 在视觉上的权重几乎一样。严肃的资源管理游戏需要在资源低位用 **高对比红警示**。
3. **`hud-resource` 完全没有 title/aria-label**（JS 实验确认）：鼠标悬停资源图标**不会**显示任何 tooltip。玩家对 Prosperity=44、Threat=28、Dev=50/100 这种数字一脸问号，却无任何即时说明。唯一有 tooltip 的地方只在左上 "Food — raw food from farms, feeds workers"，但那是 1024×768 时才偶然看到的文字，并不是 hover tooltip。
4. **核心警报和普通状态消息混在同一条绿色/红色小胶囊里**，而且会被左右截断（`Supply-chain heat lens O...` / `Emergency relief stabil...`）。一个 sim 游戏把紧急警报和"Simulation started"设计成同一个容器、还带省略号，是严重的信息优先级设计失败。

### 2.2 左侧 Build Tools + Construction 两个浮层

左侧上下叠着两个深色卡片：
- `Build Tools` —— 2 列网格，12 个按钮：Road / Farm / Lumber / Warehouse / Wall / Bridge / Erase / Quarry / Herbs / Kitchen / Smithy / Clinic。每个按钮带 16px 的图标 + 英文文字。
- `Construction` —— 显示 `Selected Tool: Road`、简短说明、`Cost: 1w`、Rules、以及动态提示（例如 `Insufficient resources.` / `Cannot build on water tile.` / `Clear the wall before building here.`）。

评价：
- 信息层级**算是这套 UI 里最合理的地方**。selected 态用蓝色描边高亮 clearly visible。
- Hover 按钮有 tooltip（例如 Kitchen 显示 "Kitchen (10) — cook food into meals; cost: 5 wood + 3 stone"）—— 这是全界面少有的合格交互反馈。
- 但 **图标严重同质化**：Road / Lumber / Erase 的小图标几乎都是棕/灰色锤头或麦穗形，远距离几乎无法仅凭图标区分。不符合 RimWorld / ONI 那种"一眼看懂"的设计语言。
- `Cost: 1w` 这种速记法对新玩家不友好（w=wood？新手没上下文）。
- 该面板宽度固定，**不允许玩家拖动、折叠、关闭**，永久占掉左侧约 260×500px 的屏幕空间，遮住了大块地图。

### 2.3 右侧浮层（Colony / Settings / Debug 三选一）

三个按钮是 **互斥 toggle**，点 Colony 开 Colony 面板、点 Settings 把 Colony 关掉换成 Settings。这是个常规设计，但：

- **Colony 面板** 分 3 块：Resources 进度条列表（Food 103 / Wood 8 / Stone 4 / Herbs 0 / Meals 0 / Tools 0 / Medicine 0）、Population（Workers 15 / Visitors 4 / Herbivores 3 / Predators 1）、再加一个用**全大写**的 role 列表（`FARM 3 / WOOD 8 / STONE 1 / HERBS 1 / COOK 0 / SMITH 1 / HERBALIST 0 / HAUL 1`）。全大写和普通大小写两种风格同屏出现，风格不统一。
- **Settings 面板**（全部展开后）一路铺出 20+ 个 slider：Target Farmer Ratio 50%、Sim Tick 30.0 Hz、Min Zoom 0.55、Max Zoom 3.20、Detail Threshold 260、Water Level 0.16、River Count 1、River Width 2.2、River Meander 0.12、Mountain 0.08、Island Bias 0.03、Ocean Bias 0.00、Road Density 72%、Settlement 78%、Wall Mode、Ocean Side、Population Control … 这**根本不是 Settings，而是开发者调参台**。普通玩家不该在主 HUD 的 Settings 按钮下看到"河流弯曲度 0.12"这种数字。
- **Debug 面板** 更夸张：World State 逐行显示 Prosperity 45.7、Threat 52.4、Survival、Weather clear (14s)、AI Mode "off / fallback (unknown, -)"、AI Env "fallback @ 60.2s | steady state"、AI Policy "fallback @ 60.2s | workers=frontier buildout"、AI Decisions "traders:warehouse circulation | workers:frontier buildout | saboteurs:economic harassment"、Events `trade caravan cooldown @ east ruined depot high pressure 1.64 contested 13` … 这就是一个**完全赤裸的 AI debug log**。再加下面 Inspector（显示 `Coord: (36, 28) | idx=2724 · Type: ROAD (1) · Passable: true · Move Cost: 0.65 · Height: 0.025 · Grid Version: 74 · Neighbors: N 1 | S 0 | W 1 | E 1`）、Event Queue、AI Decisions、AI I/O、Stress Test —— 全是研发/调优面板。

**结论**：Settings / Debug 两个 toggle **不应该默认出现在玩家 HUD 上**。这些内容该放在独立的 `?debug=1` URL query 或者隐藏快捷键。对普通玩家，看到这些等于直接把引擎剖开放在脸上。

### 2.4 底栏

底部居中一条圆角黑条，分三块：
- 中间 3 按钮：`⏸ ▶ ⏩` + `0:00` 时钟 —— 唯一比较克制、干净的地方。但用 Unicode 箭头符号 `▶` `⏩` 充当"正常速度"和"快进"，视觉几乎无差异；RimWorld / ONI 都有明显的"单三角 / 双三角 / 三三角 / ⏸"的配色区分，这里全是一模一样的白色。
- 下方一个折叠条 `Entity Focus`：点击展开后显示 "Click a worker, visitor, or animal on the map to inspect it here. No entity selected." —— 这是一个**空状态设计**，但文字占了大半屏、无图示、无颜色引导，非常 barebones。

### 2.5 Dev Telemetry 巨型底部面板

底部还隐藏着一个可以"Show Dev Telemetry"唤出的巨型横向条，分 6 列：Global & Gameplay / A\* + Boids / AI Trace / Logic Consistency / System Timings / Objective / Event Log。里面全是 `Map: Rugged Highlands (rugged_highlands) · Seed: 36459 · Grid v21 · 96x72 · Terrain: passable=91.0% road=31 water=577 walls=21 emptyBase=0 · Tuning: water=0.16 riverCount=1 ...` 这种纯文本。又是一坨研发 dump。专业玩家和 modder 会喜欢，但它**默认按钮就摆在 Debug 面板底部**，没有"这是高级内容"的任何提示。

## 三、视觉设计

### 3.1 色彩系统

- 主色：一个朴素的深蓝灰 (`#0a1420` 左右) 背景、淡蓝 (`#4da8ff` 感觉) 强调色、白色/灰白文字。这组配色本身不难看，**问题在于毫无层级**。所有面板的背景颜色、边框颜色、字体颜色几乎一样，看一眼分不出"这是信息"和"这是按钮"。
- 资源条的绿条/橙条是极窄的 2px 亮条，在深色背景上几乎看不见。
- 警告类横幅用**绿底**（"Simulation started"）和**红底**（"Insufficient resources"）区分，但文字颜色都是纯白——红绿色盲玩家基本丢失信息。
- 地图中"危险实体"用**橙/粉色圆圈**脉冲高亮，这是少数做得尚可的视觉反馈。

### 3.2 字体

- 全界面只用了一种 sans-serif（从渲染看是系统默认的 Segoe UI 或 Inter 类字体）。
- 没有字号层级：从 `Build Tools` 标题到 `FARM 3` 到底部 `0:00` 时钟字号几乎都在 11–14px 之间。
- 大量使用**全大写 + small-caps**（BROKEN FRONTIER、SEED 1337、FARM/WOOD/HERBS 列表），显得生硬。
- 说明文字和数值统统用 **浅灰 (#9aa4b2 类)**，在深色背景上对比度勉强及格；在白色背景的 tooltip 上就直接变成"浅灰字 + 深底"，可读性略差。

### 3.3 按钮/面板设计语言

- 面板全部是圆角矩形 + 深蓝灰底 + 淡灰边。风格**内部算一致**，但**过于性冷淡**，没有任何材质、纹理、icon 色彩点缀。
- 顶栏 toggle 按钮（Build / Colony / Settings / Debug / Heat Lens）——当前选中的 Build 用蓝色描边，其他用灰色描边。差异很微弱。
- Playback 控件（⏸ ▶ ⏩）选中态用淡蓝背景，但"播放速度切换"没有视觉上的分级。

### 3.4 图标

- 资源图标（Food/Wood/Stone/…）看起来像 RimWorld 那种 32×32 小 icon 再缩成 16px，渲染模糊、**无 hinting、无描边**，混在一起几乎分不出"蘑菇 / 麦穗 / 肉"。
- Build Tools 的按钮图标同样浅淡缺乏辨识度。

## 四、交互反馈

- **悬停反馈**：Build Tools 按钮有 tooltip，这是为数不多的亮点；资源条、population 行、role 列表、Debug 里大量指标**全部没有 tooltip**。玩家想知道 Prosperity 44 是高还是低、Dev 50/100 代表什么——只能猜。
- **点击反馈**：按钮点击几乎没有 press-down 动画、没有 ripple、没有声音提示（至少视觉上没有——未测音频）。
- **选中态**：被选中的工具用蓝色描边，算是能看出来，但很淡。
- **错误反馈**：报错走顶部滚动 banner，银绿/银红色一闪而过，不会停留；被压在快速变化的"Dev 50/100"旁边极容易错过。
- **建造失败反馈**：尝试在水上造路时会在 Construction 面板里加一行红字 "Cannot build on water tile."——但地图上被点击的水格没有红色闪烁或震动，没有"啊，不能建"的直观回应。
- **键盘导航**：按 L 键正常切换 Heat Lens；按 Space 可以暂停。但 Tab 键的 focus ring 在面板里乱跳，没有明显的 focus 可视化轮廓。

## 五、3D / 地图渲染

这款自称 Three.js 的殖民地 sim，从画面看更像**自上而下的 2.5D 贴图**（每个 tile 是一个顶视方块，角色是像素小人小贴图）。评测如下：

- **地形**：草地 tile 是一块纯绿方块 + 极轻的纹理细节，看不出任何地形起伏；水 tile 是浅蓝色 + 很明显的白色"波纹/花瓣"纹理（像素风），海量铺开后形成一种花布效果，非常吵闹。森林是深绿色方块，就是把绿色叠深一阶。道路/农田是土黄色，看得出是采 RimWorld 风。
- **建筑**：每个建筑贴图是一个带红/褐瓦顶、齿轮（Lumber/Smithy？）、麻袋（Warehouse？）的 16×16 或 24×24 小 sprite。**极小、几乎看不清**。全局视角看过去就是一堆灰红色格子。
- **角色**：白色的"小木偶"形状，像 14px 高的像素人，**没有任何方向面部差异**，全是同一个贴图在走来走去。远一点你根本分不清"这是 worker 还是 visitor 还是 saboteur"，只有在 Debug 里才能看到类型统计。
- **光影**：基本无光照效果。没有阴影、没有昼夜交替（虽然 state 里有 `Weather clear (14s)`，但地图表层毫无表现）、没有天气粒子。仅在 tile 过渡边界能看到一条极硬的像素楼梯。
- **缩放**：滚轮可以自由缩放（min 0.55 ~ max 3.20），**放大后 sprite 被双线性插值拉糊**（不是像素完美缩放，也不是矢量重绘）。缩放到最小后地图变成一个不到屏幕 1/3 的小方形，**四周是一片纯蓝灰背景**（#0a1420 body 色）毫无 vignette、边框、小地图、甚至没有"地图到此为止"的标识，非常廉价。

在 **3D 呈现这一维度上，本作没有任何"3D"的实感**——换成 Canvas 2D 几乎没区别。与宣称 Three.js 的期待严重不符。

## 七、响应式测试

| 分辨率 | 结果 |
|--------|------|
| 1920×1080 | 基本可用；但 Dev Telemetry 默认隐藏，右侧大量空白 |
| 1024×768 | 顶栏开始文字省略号；左侧面板仍占 260px，可用地图面积大幅缩水 |
| 800×600 | **顶栏按钮（Settings/Debug/Heat Lens）被切出屏幕**，功能丢失；不可用 |
| 600×900 竖屏 | 顶栏只剩 5 个资源（其余消失）；Build 面板铺满屏幕；几乎没有游戏画面 |

结论：**不支持 1366×768 以下分辨率**，没有任何 breakpoints 或移动端布局。这在 2026 年是不可接受的基线。

## 八、可访问性

- **无 tooltip/aria-label** 在顶栏资源（验证过 `title=""` `aria-label=null`）。
- **无 focus ring 可视化**，键盘用户难以定位。
- **色盲友好度低**：绿/红 banner、蓝色选中态，没有图形辅助。
- **字号不可调**，且浅灰字在 HiDPI 下容易变糊。
- **屏幕阅读器**：页面使用 `<details>` / `<summary>` 原生元素是个加分项（至少 Colony/Debug 分组可被朗读），但标题常常只是 "workers | ttl=24.0s | risk=0.35" 这类 AI 指标，读起来毫无意义。

## 九、与标杆产品对比

| 参考 | Project Utopia 的差距 |
|------|------------------------|
| **RimWorld** | RW 的资源图标大、颜色饱和、分类明确；底部有建筑 / 角色 / 计划三 tab，永远不会把"河流弯曲度"放进玩家菜单。Utopia 完败。|
| **Factorio** | Factorio 每个 building 都有饱和色 + 大图标 + 强烈边框，远距离可辨。Utopia 的小人/建筑几乎只有灰棕一色调，信息密度极高但辨识度极低。|
| **Oxygen Not Included** | ONI 有分层的顶栏（资源、警告、时间、天气）、明确的"Overlay"按钮阵列（Oxygen / Power / Plumbing…）、每个 overlay 都有图例。Utopia 的 Heat Lens 点了等于没点，没有图例。|
| **Frostpunk / Against the Storm** | 这些作品的暖色调、强光影、剧情叙事扑面而来，Utopia 没有任何戏剧感。|
| **Dwarf Fortress (Steam 版)** | 连 DF 现在都知道做一套 tooltip 图标+信息面板。Utopia 在信息披露方式上还不如 DF Premium。|

---

## 十、改进建议（优先级排序）

1. **P0 - 把 Debug/Settings 从玩家 HUD 剥离**：仅在 `?dev=1` 或开发快捷键 (Ctrl+Shift+D) 下显示。给玩家的 Settings 只保留 5 项：语言、音量、画质、全屏、重置游戏。
2. **P0 - 资源条重做**：放大到 ~28px 图标 + 18px 白色粗体数字 + 10px 灰色"增速/净流"。为每个资源加 hover tooltip 说明用途和涨跌趋势。低于阈值时整块用红底+闪烁提示。
3. **P0 - 响应式最低支持 1280×720**：把顶栏资源转到可滚动或自动折叠，Settings/Debug/Heat Lens 在小屏合并进"更多"菜单。
4. **P1 - Heat Lens 必须有视觉变化**：热力图需要覆盖在地图上，附带图例（红=供应过剩、蓝=饥饿）。目前点了按钮只有顶栏一闪提示不合格。
5. **P1 - 建筑和角色 sprite 重绘**：至少给主建筑加 3~4 帧的状态（工作中/空闲/损毁）和 16×16 颜色明显的分类色。
6. **P1 - 警报系统独立**：红色告警不该和普通状态消息共用 banner；需要独立的右上角弹窗+角标数字。
7. **P2 - 字体层级**：标题 20px bold / 段标题 14px medium / 数据 16px semibold / 说明 12px regular，统一一套 type scale。
8. **P2 - 开场美术**：splash 替换为有 hero 插画 / 过场动画的欢迎页，而不是裸露的 seed 号。
9. **P2 - 可访问性**：所有 interactive 元素加 aria-label；提供至少 dark-colorblind safe 配色方案。
10. **P3 - 3D 表现力**：若真使用 Three.js，应该加日夜循环、投影阴影、材质变化、天气粒子——目前 `Weather clear (14s)` 这行字在画面上零呈现。

---

## 结论

**UI 不合格。** 作为一个面向商业市场的 sim 游戏 alpha，它在**信息层级**、**视觉层级**、**响应式**、**交互承诺兑现**、**美术完成度** 五个维度都未达到付费 standard；仅在**构造面板 tooltip**和**Inspector 可挖掘程度**上展露了些许工程底子。开发团队显然把 90% 的心力投入在 ECS 系统 / AI 调度 / 地形生成参数，这在底层技术上值得尊敬，但拿出来给"第一次接触这款游戏的外部玩家"看时，扑面而来的是一张**工程师的工作台**，不是一款**游戏**。

如果要把它从 3.5/10 抬到 7/10 商业可卖的及格线，至少需要：整套 UI 重绘、响应式重写、建筑/角色美术重做、把 Debug 全部藏到开发者模式里、为所有资源/指标加 tooltip + 颜色警示。

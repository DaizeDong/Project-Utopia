## Feature Complete Release Notes

### Quick Review

Since the Beta (HW05 / v0.7.1), the project shipped the `v0.8.0 "Living World"` balance
overhaul (7 phases, 4 milestones), the `v0.8.1 "Phase 8 Survival Hardening"` patch, and
the in-progress `Phase 10` determinism hardening. The two pillars (colony-scale NPC AI,
and pathfinding + crowd navigation) are now frozen — no new mechanics are planned for
HW07.

### Final Feature Integrations (since Beta)

- **Survival-mode game loop.** Retired the 3-objective win condition; the run now ends
  only on colony loss or on reaching the day-730 cap. `DevIndex` (6-dimension composite:
  population, economy, infrastructure, knowledge, AI quality, environment) replaces
  the old objective gates as the headline metric.
- **Raid escalator.** A 6-tier `RaidEscalatorSystem` scales raid frequency and lethality
  with `DevIndex`, so late-game pressure keeps up with colony size instead of flattening.
- **Living-world ecology.** Soil salinization + farm fallow, deterministic resource
  nodes, fog-of-war with reveal radius, and building demolition recycling (M1 / M1a /
  M1b / M1c).
- **Warehouse throughput + density risk.** Deposit queue with per-tick drain cap, plus
  density-driven risk events (fire / disease) when buildings crowd (M2).
- **Carry fatigue + spoilage.** Workers slow on long hauls; food-carry has a spoilage
  timer (M3).
- **Road compounding.** Road wear degrades the speed bonus and logistics efficiency,
  so neglected networks bottleneck late-game (M4).
- **AI adaptation sweep.** 18 targeted patches across `Perceiver / Planner / Executor /
  Evaluator` — memory-stream pruning, skill-library feasibility checks, plan rollback
  on evaluator regression, deterministic fallback parity with live LLM.
- **Supply-Chain Heat Lens.** `L`-key overlay (also a HUD button) paints per-tile
  logistics heat so players can see where deliveries bottleneck.
- **Long-horizon benchmark harness.** `scripts/long-horizon-bench.mjs` runs up to 730
  in-game days with checkpoints at days {30, 90, 180, 365, 548, 730}. Wired to CI as a
  smoke job (90-day bound).
- **`deliverWithoutCarry` exploit fix.** A 7-test regression suite locks the invariant
  (deposits require carried resources).
- **Phase 8 survival hardening.** `yieldPool` lazy-init bug in farm harvest, missing
  kitchen tier in the fallback planner, salinization tuning (0.02→0.012 per-harvest,
  1800→1200 fallow ticks), fog initial radius 4→6, and demand-side growth throttle
  (FOOD_COST 6→10, pop-cap tightening). Day-365 `DevIndex` 39→44 (+12%), deaths
  512→454 (−11%).
- **Phase 10 long-horizon determinism.** `bootHeadlessSim` is now bit-identical across
  repeated boots with the same seed + preset. `WeatherSystem`, `Grid` fertility init,
  `WildlifePopulationSystem`, `pathBudget.maxMs`, `BenchmarkPresets.applyPreset`, and
  all `randomPassableTile` callers were audited to route through `services.rng` instead
  of `Math.random`. 3 new regression tests (`test/long-horizon-determinism.test.js`)
  lock the contract.

### Test / Benchmark Status

- **870 tests** across 110 files: 868 pass, 2 intentional skip, 0 fail
  (`node --test test/*.test.js`).
- **Long-horizon bench (seed 42 / temperate_plains, 90 days).** Two cross-process
  invocations of `scripts/long-horizon-bench.mjs` produce byte-identical
  `outcome / daysCompleted / devIndex / survivalScore` (only wall-clock differs). Previously
  these diverged by 10–40% due to wall-clock-driven `Math.random`.
- **CHANGELOG.md** carries one `### Phase` entry per sub-phase (136 entries across
  `0.7.0 → Unreleased`) so every behavior change is traceable to a dated commit.

### Known Issues (into HW07)

- **DevIndex balance gap.** Day-365 `DevIndex` on seed 42 / temperate_plains sits
  around 37 vs. the target ≥ 70. Traced in the progress tracker to structural issues
  (`BuildAdvisor` priority, `INITIAL_RESOURCES`, carry-eat bypass); punted to the HW07
  optimization sprint.
- **Live LLM mode** still requires a local API key; the deterministic fallback stays
  stable without it. When running without a key, a persistent warning banner
  (`AI proxy is unreachable; running fallback`) surfaces on the HUD with no player-facing
  explanation.
- **Wildlife balance** is conservative and tends to settle at a low population floor.
- **`vendor-three`** production chunk is still large — scheduled for the HW07
  perf/optimization pass.
- **Early-game economy cliff.** On a fresh boot with default seed, food trends negative
  in the first 2–3 minutes before the first farm harvest lands, which makes the new-
  player window unforgiving. Subject 3 and 5 both hit this.
- **Developer Telemetry / Settings exposure.** The Debug Telemetry panel and the
  world-generation sliders in Settings are visible by default, which reads as an unfinished
  editor build to non-technical players. Needs a `?dev=1` / hotkey gate.
- **No audio at all.** The build ships with zero `<audio>` elements (UI click, build,
  alarm, ambient). This is a known omission from the original plan, kept out for the
  HW06 freeze and moved to HW07.
- **Settings panel can silently reset the world.** Changing world-generation sliders
  mid-run reroutes through `applyPreset` and wipes colony state without a confirm dialog.
- **`Score` incentive inversion.** With the current survival-score formula, "do nothing"
  accumulates score faster than active building on short (<5 min) runs, because
  deaths/drownings penalize aggressive expansion.

## User Study Results

### Disclosure & Methodology

**I did not recruit 5 human classmates. Instead, I commissioned 10 independent LLM
agents (via the Claude Agent SDK, each driving a real headless browser through
Playwright) to play the Feature Complete build.** This is a deliberate substitution,
not an omission. My reasoning:

1. **Domain expertise, on tap.** The class cohort around me is dominated by non-gamers.
   A RimWorld veteran who can compare my warehouse throughput to RimWorld's hauling
   job-graph, or an indie-game critic who can name five peer titles in the colony-sim
   space, is extremely hard to find in a 15-minute hallway recruit. LLM agents can be
   prompted into these specialist personas on demand and deliver notes at that level.
2. **Persona diversity in a single pass.** Real peer playtests tend to converge on the
   same "I'm a CS student who plays a bit" profile. The agent pool covered five
   genuinely different player archetypes (RimWorld veteran, Stardew/AC casual,
   speedrunner/optimizer, roleplayer, indie-game critic) plus five cold-critic rubrics
   (onboarding, playability, UI, mechanics+content, innovation). That is 10 distinct
   lenses in one afternoon, which no in-person recruit pipeline would reproduce.
3. **Throughput and depth.** Each agent ran an uninterrupted 1–3-hour session and
   produced a ~10–20 KB structured review (scores per axis, reproduction steps, DOM
   audits, even `document.querySelectorAll('audio').length === 0`). A 15-minute think-
   aloud with a classmate yields maybe a paragraph of useful signal. On a per-finding
   basis the agent reviews are an order of magnitude denser.
4. **Reproducibility.** The agent prompts and raw reviews are checked into
   `assignments/homework6/Agent-Feedback-Loop/`, so anyone can re-run, re-seed, or audit
   the study. Human testimony cannot be re-run.
5. **No social-desirability bias.** Human classmates will soften criticism of a peer's
   project. Every agent review I got back was unsparing; the summary average was
   3.1 / 10.

Limitations I do not gloss over: LLM agents cannot feel frustration, bored, or "in the
flow"; they cannot detect subjective-but-real affordances like "this is cute" or
"this music fits". For the usability and UI-correctness axes that this assignment
actually targets, however, the agent panel is strictly superior to what I could
assemble from 5 classmates.

Each agent was:

1. Given only the web demo URL and the one-sentence premise ("colony-sim where you
   build and survive on a procedural map"). **Source code and docs were withheld.**
2. Asked to drive the build via Playwright for an extended session and narrate its
   reasoning as it went (the think-aloud protocol translates cleanly to agent
   narration).
3. Asked to deliver a structured post-play review: scores per axis, reproduction
   steps for issues, intuitiveness (1–10), visible glitches, and any perf notes.

### Iteration Pipeline (Round 0 → HW07)

This playtest is **Round 0** of a continuous `Agent-Feedback-Loop` I am running on
the project. Starting in HW07 each iteration will cycle through four stages:

| Stage | Actor (agent type)                  | Input                   | Output                              |
|-------|-------------------------------------|-------------------------|-------------------------------------|
| A Review    | 10 × Reviewer (fixed personas, no code access) | Live build URL    | `RoundN/Feedbacks/` + `summary.md`  |
| B Plan      | 10 × Enhancer (one per feedback, full repo + live build access) | Feedback | `RoundN/Plans/` + `summary.md` (priority-ordered) |
| C Implement | 1 × Coder (Implementer, serial per plan, commits one at a time) | Plan summary | `RoundN/Implementations/*.commit.md` |
| D Validate  | 1 × Coder (Debugger/Tester)         | Stage C HEAD            | `RoundN/Validation/test-report.md`  |

Iteration continues until (a) Stage A summary has ≤ 2 P0 findings, (b) mean score
rises from Round 0's 3.1 / 10 to ≥ 7.0 / 10, and (c) two consecutive rounds surface
no new P0 issues. At that point I take over for a 30-minute human playthrough to
verify the agents have not overfit their own rubric.

**Artifacts (all checked in)**:
- `assignments/homework6/Agent-Feedback-Loop/PROCESS.md` — full pipeline spec +
  Round 0 execution log (timeline, failures, cost model, prompt design principles)
- `assignments/homework6/Agent-Feedback-Loop/Reviewers/*.md` — 10 reviewer prompt
  constants (each with YAML frontmatter, reusable across all rounds)
- `assignments/homework6/Agent-Feedback-Loop/Enhancers/enhancer.md` — single enhancer
  prompt template; 10 subagent instances per round consume it
- `assignments/homework6/Agent-Feedback-Loop/Round0/Feedbacks/` — the 10 reviews +
  2 summaries that produced the table above

See `PROCESS.md` for the full spec; the summary here is intentionally terse so the

## Action Items for HW07

1. **Onboarding overlay.** Add a first-run 3-step tooltip sequence explaining: (a) the
   colony plays itself, (b) the build palette location, (c) how to read `DevIndex`.
2. **HUD tooltips.** Hover-tooltips on `DevIndex`, `prosperity`, `threat`, and the
   raid-tier indicator, with one-sentence explanations.
3. **Build queue.** Allow click-and-drag / shift-click to queue multiple placements of
   the same building type, addressing the RimWorld-player feedback.
4. **Raid-spawn perf pass.** Profile the stutter on first raid spawn (subjects 2 & 5);
   likely candidates are the `RaidEscalatorSystem` agent-spawn burst and the Three.js
   geometry-instance warmup.
5. **`vendor-three` chunk split.** Finish the Vite chunking pass so first-load
   time on the web demo drops; this is the residual HW05 known issue and affects
   first-impression on slower laptops (subject 3).
6. **Dev-panel gating.** Move the Debug Telemetry panel and world-gen sliders behind a
   `?dev=1` query flag (or `Ctrl+Shift+D` hotkey). Ship a player-facing Settings with
   audio / graphics / keybinds only.
7. **Worker detail redesign.** Replace the FSM / A* / policy dump in Entity Focus with a
   player-readable card (name, role, hunger bar, current task, recent events). Keep the
   raw dump behind the dev flag.
8. **Settings-change confirmation.** Any world-gen slider change mid-run must prompt
   "This will reset the colony. Continue?" — surfaced by the roleplayer persona after a
   30-minute session was wiped.
9. **Survival-score fix.** Re-weight the score formula so passive-idle runs cannot
   outscore active building; either add a build-milestone bonus or penalize zero-action
   ticks after day 3.
10. **Audio pass.** Ship a minimal SFX set (UI click, build placement, raid alarm,
    death) + one ambient loop. Zero-audio is the single most-cited immersion gap across
    both review tracks.

The first technical pillar of Project Utopia is **NPC AI / Behavior Trees with runtime LLM assistance**. ## Secondary Pillar Integration

### Pathfinding & Navigation

- Broken frontier routes, depot targets, traffic hotspots, and weather fronts now render directly on the map through a pulsing **pressure-lens overlay**, making route repair and logistics decisions readable in-world rather than only through the inspector.
- **Worker speed differentiation** — Each worker has a unique `speedMultiplier` preference (0.9–1.15×) applied to A* path-following, creating observable individuality in movement.
- **Bridge tile** (v0.5.4) — A new passable tile placed over water, connecting fragmented islands on archipelago maps. ColonyDirector auto-bridges water gaps when needed.

### Swarm / Boids / Ecology Pressure

- Wildlife pressure hotspots are now visualized in the same map-space language as logistics and weather pressure.
- Herd motion, route congestion, and ecological pressure feel like interacting systems competing for the same frontier space.
- **Congestion and traffic systems** — Workers sharing tiles generate congestion events; roads reduce movement cost and create natural traffic flow.

### Visual Cohesion

- **Scenario-aware atmosphere** — Sky, fog, and lighting respond to scenario family, weather, spatial pressure, and win/loss state. Storms darken the scene; droughts warm it; victory/defeat shift the mood.
- **Procedural textures** for all 14 tile types (including 5 processing buildings and bridge) with tint, roughness, and emissive profiles.
- **Icon-rich HUD** — Pixel-art resource icons, colored progress bars, urgency highlights, and in-viewport speed/timer controls replace the old plain-text status bar.

## Performance Check

HW02 target: ~60 FPS in browser with ~200 total entities.

- Atmosphere changes lerp a small number of light/fog values from existing state.
- Pressure-lens recomputation is signature-gated, rerunning only on relevant state changes.
- Processing buildings use per-tile cooldown timers with no per-frame allocation.
- All 22 benchmark scenarios complete without NaN, negative resources, or system errors.

Validation:

- `npm test` → 335 tests passing, 0 failures
- `npm run build` → production bundle builds cleanly

## Known Issues

1. Live-LLM validation depends on a valid local API key and reachable proxy. The project remains safe in deterministic fallback mode when unavailable.
2. SpatialLayout benchmark scores remain constrained by map template design (farms placed far from warehouses by template). Would require map generation changes to fundamentally improve.
3. The production bundle contains a large `vendor-three` chunk — functional but an optimization target for final submission.
4. Wildlife herds stabilize at a low population floor rather than a richer target; predator–prey equilibrium is conservative.

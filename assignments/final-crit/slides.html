What it is.

Premise

  • Player builds infrastructure + sets priorities
  • Agents — workers, visitors, predators, herds — execute autonomously on a 96 × 72 tile grid
  • Every living entity is a simple sphere; the world is tile-based; aesthetic is intentionally minimalist
  • The system reacts; you steer

Why simple spheres?

From the original spec (a1.md): the goal is to make the AI visible and interpretable. Reducing visual complexity lets you watch flowing convoys, scattering herds, traffic patterns reshaping as you build.

PriorityFSM — flat, priority-ordered, preemptable.

BT vs FSM — why FSM won

  • BT asks "what to do given conditions?"
  • FSM asks "what transitions can fire from current state?"
  • Our problem is transition-driven: a HARVESTING worker who sees a saboteur 2 tiles away needs to preempt instantly
  • FSM expresses preemption as a priority-1 transition — one line

12 worker states

DELIVERING → DEPOSITING | IDLE

The dispatcher — 125 LOC

tick(entity, state, services) {
  const fsm = entity.fsm;
  for (const t of transitions[fsm.state]) {
    // priority-ordered; first match wins
    if (t.when(entity, state, services)) {
      this.#exit(entity, state);
      fsm.state = t.to;
      this.#enter(entity, state);
      break;
    }
  }
  states[fsm.state].tick(entity, state);
}

Two-layer AI — LLM strategy over FSM execution.

┌──────────────────────────────────────────┐ │ strategic director · gpt-5.4-nano │ │ reads world state (JSON summary) │ │ emits structured plan: │ │ [ build quarry @90, │ │ recruit BUILDER, │ │ reassign role X to GUARD ] │ └──────────────┬───────────────────────────┘ │ intents · 0.5s sim-time gate ┌──────────────▼───────────────────────────┐ │ build proposer · per-system registry │ │ → enqueues blueprints │ └──────────────┬───────────────────────────┘ │ ┌──────────────▼───────────────────────────┐ │ priority FSM · per-worker execution │ │ workers pick up nearest available │ │ blueprint via SEEKING_BUILD state │ └──────────────────────────────────────────┘

Why this works

  • LLM never moves a worker directly — it just changes priorities
  • If LLM offline, a deterministic fallback director emits the same plan-shape from rule-based heuristics
  • Game runs cleanly without network — no degradation
  • Sim is fully deterministic: seeded RNG + fixed system order — 2060+ tests cover the AI surface without LLM calls

The challenge — paths through a living world.

Workers walk on a 96 × 72 = 6912-tile grid. The path must:

  • Avoid impassable terrain (water · walls · mountains)
  • Honor realistic costs (mountains slow · roads fast · bridges expensive but valuable)
  • Avoid collision with other workers and animals — without breaking the path
  • Adapt to fog of war — autopilot can't build on tiles it hasn't revealed
Two layers, classical: A* for the global route + Boids for local steering.
The interesting work is where they meet.

Dual-search A* — amortized bridge interleave.

Problem

  • WATER impassable → planner refuses to cross rivers, even when bridges would save 20 detour-tiles
  • WATER low-cost → planner routes through water everywhere, oblivious that bridges aren't built yet

Solution

Run A* twice. Score by cost amortized over expected traffic.

function planRoadConnections(grid, from, to) {
  const noBridge   = roadAStar({ allowBridge: false });
  const withBridge = roadAStar({ allowBridge: true  });

  const AMORT = 50;  // expected round trips/route lifetime
  const sNo   = noBridge.length;
  const sWith = withBridge.length +
                (withBridge.bridgeCount × BRIDGE_COST) / AMORT;

  return sWith < sNo ? withBridge : noBridge;
}

Result

  • 3-tile bridge that saves 20 detour-tiles × 50 trips = clear win → build
  • 1-tile bridge that saves 2 detour-tiles → not worth, walk around
  • BridgeProposer scans 8-tile shoreline pairs, can propose multi-tile bridge sequences (not just 1-tile pinch points)

Boids on path — dampened for convoys.

Problem

Pure Boids around a moving convoy fights A*: separation pushes, then path-following snaps back. Visible result: jitter.

Fix — context-aware separation weight

const sepWeight =
  (entity.path && entity.pathIndex < entity.path.length)
    ? 0.35   // worker on path: dampen, A* dominates
    : 1.0;   // animal/no path: full Boids
  • Workers in transit form flowing convoys — visible single-file along roads
  • Animals (deer, predators) keep full Boids → scatter organically
  • Same Boids implementation; one weight does both

The pillars compose — one system, two angles.

① LLM decides — "build a quarry near the stone cluster" [ pillar 1 ] ② BuildProposer checks visibility · stone is fogged [ pillar 2 ] → sets state.ai.scoutNeeded = true ③ Worker FSM's IDLE.tick reads scoutNeeded [ pillar 1 ] → calls pickFogEdgeTileNear() [ pillar 2 ] ④ Workers reveal stone tiles by walking through fog [ pillar 2 ] → BuildProposer queues quarry blueprint ⑤ BUILDER's SEEKING_BUILD state [ pillar 1 ] → runs A* for path to site [ pillar 2 ] ⑥ On the road · Boids separation = 0.35× · convoy flows [ pillar 2 ] ⑦ Predator appears mid-walk · COMBAT_PREEMPT fires (priority 1) [ pillar 1 ] → FSM transitions to FIGHTING immediately ⑧ Combat resolved · FSM returns to BUILDING · resume A* path [ pillar 1+2 ]
The pillars aren't separate features. They are the same system seen from two angles — the LLM-driven strategy layer and the autonomous-execution layer pull on the same FSM/path/Boids rope.

What I learned.

Three lessons

  • Spec fidelity isn't the goal — design fidelity is. a1 said Behavior Trees; I shipped FSM. Same intent, better fit — the "autonomous interpretable agents" intent is what mattered.
  • Telemetry first, optimization second. Lost 2 rounds chasing FPS bugs that turned out to be Playwright headless RAF capping at 1 Hz. Measure your measurement before measuring your subject.
  • AI agents need hard guardrails, not clever prompts. Across 13 rounds + 140 commits — frozen scope, rollback anchors, file-ownership tracks made agents trustworthy. Constraints > cleverness.

AI-coding insights

  • LLM was best as a reviewer — one designs, another implements, a third audits. Caught issues single-agent runs would rationalize away.
  • Most surprising: an agent bisected a 4-commit regression — ran benchmarks at every commit between R8 and R10, found a 60% DevIndex drop, traced it to one line of phase-offset math.
